# Roadmap: Movie-Experts Suite v2 — 短剧/微电影创作专家增强

**Project:** RAG-augmented movie-expert skill suite (MESV2)
**Core Value:** 每个 movie-expert skill 都能用检索增强的方式调用行业知识库,让 AI 生成的短剧/微电影在专业度上接近人类创作者水平。
**Granularity:** standard
**Mode:** yolo (auto-advance, parallelization on)
**Model profile:** quality

Derived from `.planning/research/SUMMARY.md` Recommended Build Order. Seven phases; Phase 0 is a non-negotiable blocker gate. ≤ 2 GLM-heavy skills in flight at once per phase.

## Phase Details

### Phase 0: AUDIT + Eval Skeleton (BLOCKER GATE)
**Goal:** Eliminate phantom references in all 14 existing SKILL.md files; capture a pre-refactor baseline; build the eval harness skeleton so every subsequent "improvement" claim is statistically defensible.
**Mode:** mvp
**Depends on:** Nothing (first phase; BLOCKER GATE — nothing else starts until this passes)
**Requirements:** FOUND-01, FOUND-02, FOUND-03, FOUND-04, FOUND-05, FOUND-06, FOUND-07, FOUND-08, FOUND-09, EVAL-01, EVAL-02, EVAL-03, EVAL-04 (ablation capability), EVAL-08, EVAL-09
**Success Criteria** (what must be TRUE):
  1. `scripts/verify_skill_references.py` exists and reports zero phantom references (no `wan22_video`, no fabricated "168K tokens", no FLUX 1.x sampler params) across all 14 existing SKILL.md files
  2. `_eval/baseline/<expert>/SKILL.md` snapshot exists for all 14 existing experts, tagged `eval-baseline-v1`
  3. `_eval/runner.py` produces valid position-swap (both A/B and B/A orderings) comparison output on at least one sample expert
  4. `GAP-REPORT.md` exists for all 14 experts documenting phantom-ref findings, knowledge gaps, prompt weak points, stale metrics, missing refs topics
  5. `_shared/glossary.md` skeleton published with core EN↔CN term dictionary (运镜 / hook / 卡点 / 爆款 / 男频 / 女频 etc.)
**Plans:** 5/4 plans complete
- [x] 00-01-PLAN.md — Phantom-ref detector (verify_skill_references.py + allowlist)
- [x] 00-02-PLAN.md — Baseline snapshots × 14 (snapshot.py + provenance)
- [x] 00-03-PLAN.md — Eval harness skeleton (MT-Bench position-swap runner + judge prompt)
- [x] 00-04-PLAN.md — GAP-REPORTs × 14 + phantom cleanup + _shared foundation docs

**Mode:** mvp
**Depends on:** Phase 0 (audit baseline + harness skeleton; related_skills graph conventions established)
**Requirements:** COMPLI-01, COMPLI-02, COMPLI-03, COMPLI-04, COMPLI-05, COMPLI-06, COMPLI-07, COMPLI-08, COMPLI-09
**Success Criteria** (what must be TRUE):
  1. `skills/movie-experts/compliance_marketing/SKILL.md` exists with bilingual content (EN structure + CN descriptive prose)
  2. 5 reference files exist covering 网信办 AI 标识办法 + AI 漫剧 备案 regime + 3 platform specs (抖音 / 快手 / 微信小程序剧) + 爆款 elements vs 审核风险 catalog
  3. All COMPLI refs carry `verified_date: YYYY-MM` stamp with quarterly refresh cadence documented
  4. Expert integrated into related_skills graph with bidirectional edges to/from screenplay, editor, hook_retention, style_genome, drawer
  5. 5 eval prompts covering diverse compliance scenarios runnable through `_eval/runner.py`

**Requirements:** HOOK-01, HOOK-02, HOOK-03, HOOK-04, HOOK-05, HOOK-06, HOOK-07, HOOK-08, HOOK-09

### Phase 3: Top-4 Existing Experts RAG
**Goal:** Deep-refactor the four highest-leverage existing experts (screenplay, editor, colorist, style_genome) with curated RAG refs, revised thresholds, refined metrics, and statistically defensible before/after comparison.
**Mode:** mvp
**Depends on:** Phase 0 (baseline snapshot + harness), Phase 2 (HOOK's emotion_curve schema extension absorbed natively by screenplay refactor)
**Requirements:** REFACTOR-01, REFACTOR-02, REFACTOR-03, REFACTOR-04, REFACTOR-05, REFACTOR-06, REFACTOR-07, REFACTOR-08
**Success Criteria** (what must be TRUE):
  1. Each of 4 experts has 4-6 curated refs, each containing ≥1 concrete heuristic/number/rule NOT in base model training (no Wikipedia summaries)
  2. Each refactored SKILL.md has provider-agnostic RAG invocation block + revised quality thresholds + refined metrics; frozen `expert_id` values preserved unchanged
  3. 3 eval prompts per expert (12 total) runnable through `_eval/runner.py`
  4. Ablation comparison produced per expert (old-no-refs vs new-no-refs vs new-with-refs)
  5. Post-refactor eval comparison vs Phase 0 baseline produced for all 4 experts
**Plans:** TBD
**Constraint:** ≤ 2 experts in flight at once (GLM cap). Sequential screenplay → editor → colorist → style_genome preferred.
