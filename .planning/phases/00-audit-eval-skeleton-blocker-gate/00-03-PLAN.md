<!-- MT-Bench position-swap pattern (per CONTEXT.md decisions) -->
For each prompt p and judge j:
  1. Call test model twice (once per condition A=baseline, B=candidate-refactored)
  2. Present to judge j in BOTH orderings: [A,B] and [B,A]
  3. Judge returns decision: "A" | "B" | "tie"
  4. If ordering-1 says A and ordering-2 says B (or vice versa) → tie (disagreement resolves to tie)
  5. Final verdict per (prompt, judge): one of {A_wins, B_wins, tie}

<!-- Ablation capability (EVAL-04): runner must accept N conditions, not just 2 -->
Conditions for Phase 0 skeleton: support 2-or-more condition labels (configurable). Phase 3 will use 3 conditions: old-no-refs, new-no-refs, new-with-refs. Skeleton supports arbitrary list, pairwise comparison.

<!-- Judge prompt 4 dimensions (per CONTEXT.md specifics) -->
1. industry_accuracy (does output match real-world 短剧 craft?)
2. professional_depth (concrete heuristics vs hand-wavy advice?)
3. actionability (can a creator act on this?)
4. language_quality (bilingual consistency, no awkward phrasing)
