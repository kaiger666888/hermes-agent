## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-52-01 | Tampering | Operator-supplied YAML with malicious content (e.g. YAML tags triggering code execution) | mitigate | Use `yaml.safe_load` (NOT `yaml.load`) — refuses arbitrary tags by default |
| T-52-02 | Tampering | Path traversal via filename (e.g. `../../etc/passwd.agent.yaml`) | accept | glob("*.agent.yaml") at fixed parent dir; Path normalization handles `..` — cannot escape the agents/ dir via glob alone |
| T-52-03 | Spoofing | Filename/name mismatch (symlink attack where `name: alice` lives at `bob.agent.yaml`) | mitigate | load_one_agent_yaml explicitly verifies `data["name"] == filename.stem` and raises RegistryValidationError on mismatch (SC#1 acceptance) |
| T-52-04 | Tampering | Undeclared fields in agent YAML (forward-compat / polyglot attacks) | mitigate | `additionalProperties: false` in agents-schema.yaml — jsonschema rejects unknown fields |
| T-52-SC | Tampering | (No package installs in this plan — all deps already pinned) | N/A | N/A |
