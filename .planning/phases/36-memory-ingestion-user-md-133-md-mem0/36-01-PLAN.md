mem0 v2 API response unwrap pattern (from Mem0MemoryProvider._unwrap_results):
- `client.get_all()` may return `{"results": [...]}` (v2) or `[...]` (v1). Unwrap dict.get("results", []) if isinstance dict.
- Each memory entry: `{"id": "...", "memory": "...", "metadata": {...}}`

Idempotency contract:
- Each ingested memory's `metadata` MUST include `content_hash` (SHA-256 hex of file body) + `source_path` (openclaw absolute path) + `migrated_at` ("2026-06-25").
- Before adding: enumerate `client.get_all(filters={"user_id": ...})`, build a set of existing `content_hash` values from each entry's metadata. Skip files whose hash is already in the set.
- mem0 `client.add` signature: `client.add(messages=[{"role": "user", "content": body}], user_id=..., agent_id=..., metadata={...})`. Verify the `metadata=` kwarg is accepted by the installed mem0 version; if not, fall back to including the hash inside the content body as a header comment and document the deviation in INGESTION-NOTE.md.
