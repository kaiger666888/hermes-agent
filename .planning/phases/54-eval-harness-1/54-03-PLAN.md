<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fixture JSON → canary | 6 synthetic records parsed by json; malicious fixture could exploit parser |
| GLM claim-check response → flag | LLM returns "yes"/"no" parsed as boolean; malformed response must not crash canary |
| canary report → audit chain | Report summary appended to curator_audit sha256 chain — chain integrity load-bearing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-54-07 | Tampering | fixture JSON files | mitigate | use `json.load` (not `eval`/`yaml.unsafe_load`); fixture files are checked-in, version-controlled |
| T-54-08 | Denial of Service | LLM claim-check malformed response | mitigate | wrap LLM call in try/except; on parse failure, default to `claim_supported=False` + log warning (fail-safe: flag the record rather than accept it) |
| T-54-09 | Spoofing | audit chain append | mitigate | use existing `curator_audit.append_audit` sha256-chained path (already load-bearing in v6.0); do not invent new audit writer |
| T-54-10 | Repudiation | canary acceptance verdict | mitigate | verdict + per-record flags persisted in JSON report + audit chain entry; both reproducible from fixtures |
| T-54-11 | Information Disclosure | fixture memory content | accept | synthetic content, no PII, no operator data |
| T-54-SC | Tampering | no new package installs | accept | uses stdlib `math` + `json` + existing `auxiliary_client`/`curator_audit`/`glm_concurrency_guard`; zero new deps |
</threat_model>
