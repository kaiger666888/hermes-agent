## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scenario YAML → runner | YAML files parsed by ruamel/PyYAML; malicious YAML could exploit loaders |
| LLM judge → score | Judge output parsed as float; malformed judge response must not crash runner |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-54-01 | Tampering | scenario YAML loader | mitigate | use `yaml.safe_load` (not `yaml.load`); never `yaml.unsafe_load` |
| T-54-02 | Information Disclosure | fitness_trend.jsonl | accept | stored under `~/.hermes/eval/` which is operator-owned local path; no PII in scenarios |
| T-54-03 | Denial of Service | LLM judge prompt | mitigate | clamp max_tokens on judge call + timeout=30s; on malformed judge response, fall back to score=0.0 + log warning |
| T-54-SC | Tampering | no package installs in this plan | accept | zero new packages (PyYAML + ruamel.yaml already pinned per Phase 52); no `[SLOP]`/`[SUS]` packages introduced |
</threat_model>
