## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator env → adapter live path | Env-key presence gates the live HTTP call. V9-FUTURE-01 implementer must add: HTTPS verification, timeout, retry, response schema validation. |
| variant_id → URL construction | Adapters may interpolate variant_id into URLs (V9-FUTURE-01 concern; v9.0 stubs don't). Must use urllib.parse.quote. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-05 | Information Disclosure | Adapter logs env var value | mitigate | Adapters use `os.environ.get(env_key)` only for the bool check; never log the value. Test_enforced: assert no logging call includes the env var value. |
| T-42-06 | Spoofing | Live HTTP response (V9-FUTURE-01) | accept | Out of scope for v9.0 (stubs raise NotImplementedError). V9-FUTURE-01 implementer must add TLS verification + response schema validation. Documented in stub message. |
| T-42-07 | Tampering | variant_id → URL interpolation | mitigate | v9.0 stubs do NOT interpolate (raise before HTTP path). V9-FUTURE-01 message documents the requirement. |
| T-42-08 | Repudiation | Adapter fetch audit trail | accept | V9-FUTURE-01 concern; v9.0 stubs emit no side effects. |
| T-42-SC | Tampering | (no package installs) | accept | N/A — pure stdlib + httpx (already Hermes core). |
