<objective>
Add prompt-length-aware endpoint routing to `agent/auxiliary_client.py::call_llm` so long-prompt synthesis calls (≥4096 input tokens) route to `open.bigmodel.cn/api/anthropic` (anthropic-compat, no 30s cap) while short-prompt panelist calls stay on `z.ai/api/coding/paas/v4` (lower cost).

Purpose: Eliminate the v11.0 SC#2 synthesis 5x retry storm — z.ai coding plan's 30s timeout trips on the ~5K-input-token synthesis prompt, then openai-SDK retries 5x before falling back to anthropic-compat. v11.0 measured 490s total round-table latency; ~250s of that was the retry storm on the synthesis call. This plan drops the same workload to <240s by routing natively to the anthropic endpoint for long prompts.

Output: 1 helper function + 1 wire-in edit in `call_llm` + unit/integration tests + cli-config.yaml.example documentation block.
</objective>
