## Goal

把 Bob Eberle SCAMPER 7 动词变体引擎**叠加**(stacked,不替代)到 `style_genome.style_blend` 子任务上,让 `hook_retention` 消费 SCAMPER × 5 爆款公式交叉表得到 35 个 hook 变体种子。**Phase 21 同时是 v4.0 milestone 的 close-out phase** —— 把 Phase 19/20/21 三个新 ref 同步到 README.md corpus tree + skills-mapping.yaml sign-off,完成 v4.0 documentation 收口。

SCAMPER 的角色边界:**变体引擎(variation engine)**,不是分类系统(classification system)。auteur-theory + genre-dna-taxonomy 是分类系统(决定 director tier / genre 5D 区间);SCAMPER 在分类结果确定后,系统化生成 7 个 style blend 候选供下游或用户挑选。SCAMPER 与 style_blend 是**叠加关系**(style_blend 输入 → SCAMPER 展开 → 7 候选输出),不是替代。

Phase 21 不创建新 expert_id,只在 `skills/movie-experts/` 下做 5 处增量 + 2 处 cross-cutting documentation(1 新 ref + 2 SKILL.md body patch + 1 glossary append + 1 README inventory update + 1 skills-mapping sign-off append)。

## File-level Plan

### F1: NEW `skills/movie-experts/style_genome/references/scamper-variations.md`

**Target length:** 300-450 行(SCAMPER 7 动词 × 5 基因组合 = 35 个变体配方,每个配方至少 8-12 行展开,加上 LLM prompt 模板 + output schema;对齐 Phase 19 snowflake-method.md 279 行 + Phase 20 e-konte-format.md 371 行密度)。

**Structure:**
1. Header + Source + Copyright(Fair Use paraphrase + Eberle 1971 + Osborn 创意清单溯源 + LICENSE.md 引用,对齐 auteur-theory.md / genre-dna-taxonomy.md / snowflake-method.md §1-12 行模式)
2. Summary — SCAMPER 为什么填补 style_blend 单一输出的变体塌陷
3. ## SCAMPER Method — Origin & Definition — Bob Eberle 1971 系统化 + Osborn 创意清单(brainstorming)的溯源
4. ## The 7 SCAMPER Verbs — 7 动词详解
   - S — **Substitute** 替代
   - C — **Combine** 组合
   - A — **Adapt** 适配
   - M — **Modify/Magnify/Minify** 修改/放大/缩小
   - P — **Put to other use** 另作他用
   - E — **Eliminate** 消除
   - R — **Reverse/Rearrange** 反转/重排
   - 每动词:CN/EN 双语定义 + 变体操作目标(style 5D 维度) + 短剧 / 微电影 适用场景
5. ## SCAMPER × Style Genome Integration — 叠加在 style_blend 之上
   - 输入:style_genome.json(5D vector + director/genre reference)
   - 处理:7 动词各产生 1 个候选 style_blend 变体
   - 输出:7 个候选 style_blend_protocol.json 数组 + 推荐分数(novelty × feasibility × alignment)
   - **叠加不替代声明:** SCAMPER 是变体引擎,不是分类系统(auteur-theory / genre-dna 才是分类系统);SCAMPER 不动 director tier 判定,不动 genre 5D 区间
6. ## 35 Variation Recipes — 7 Verbs × 5 Gene Combinations — load-bearing 核心表
   - **5 基因组合(gene combinations):**
     1. **C1:** genre=男频复仇 × mood=压抑-释放 × pacing=1.5s × cast=单主角 × runtime=90s 单集
     2. **C2:** genre=女频甜宠 × mood=误解-揭露-圆满 × pacing=2s × cast=男女双主角 × runtime=60s 单集
     3. **C3:** genre=悬疑烧脑 × mood=多峰反转 × pacing=2.5s × cast=多角色 × runtime=180s 单集
     4. **C4:** genre=微电影文艺 × mood=内省-顿悟 × pacing=3-4s × cast=单主角 × runtime=5-15min
     5. **C5:** genre=漫剧拟人化 × mood=情谊-冒险 × pacing=1.5-2s × cast=拟人主角 × runtime=120s 单集
   - 7 动词 × 5 组合 = 35 配方,每个配方含:
     - recipe_id(如 `S-C1` / `C-C3` / `R-C4`)
     - 输入 style_genome 摘要
     - 变体动作(动词 × 操作目标,如 "Substitute color palette from cool→warm" / "Reverse rhythm from slow→fast cut")
     - 输出候选 style_blend JSON 摘要
     - 适用场景(60s/90s/180s × 男频/女频 × 平台)
     - 反指示 / 何时不用此配方
7. ## LLM Prompt Template — 1 通用 + 7 动词专用 prompt
   - 通用 prompt:输入 style_genome.json → 输出 7 候选
   - 7 动词专用 prompt:每动词 1 个 prompt,聚焦该动词的变体生成
8. ## Output Schema (`scamper_variants.json`) — 7 候选数组的 JSON schema
   - 字段:`recipe_id` / `verb` / `original_genome` / `modified_genome` / `modification_summary` / `novelty_score` / `feasibility_score` / `alignment_score` / `recommended_use_cases`
9. ## Integration with auteur-theory + genre-dna-taxonomy
   - SCAMPER 不替代 auteur-theory 的 tier 判定(Pantheon/Modern Auteur/Operator)
   - SCAMPER 不替代 genre-dna 的 5D 区间表
   - SCAMPER 在 auteur-theory + genre-dna 输出确定的 style_genome 基础上展开 7 个变体候选
10. ## Anti-Patterns / What NOT to do
    - 不要把 SCAMPER 当作分类系统(它是变体引擎)
    - 不要让 SCAMPER 替代 director tier 判定(那是 auteur-theory)
    - 不要让 SCAMPER 改写 genre 5D 区间(那是 genre-dna)
    - 不要把 7 候选全部应用(用户/下游应基于推荐分数挑选)
11. ## License — Fair Use paraphrase 声明 + LICENSE.md 引用

**License status:** Fair Use paraphrase(SCAMPER 7 动词是创意方法论通用术语,Eberle 1971 系统化 + Osborn 创意清单溯源)。本 ref 不复制 Eberle 原书内容,只 paraphrase 7 动词定义 + 自创 35 配方与短剧适配。沿用 snowflake-method.md / e-konte-format.md / auteur-theory.md Fair Use 模式。
