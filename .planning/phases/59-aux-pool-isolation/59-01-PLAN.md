<objective>
Build POOL-01: separate auxiliary credential pool that isolates auxiliary traffic (round table, memory_compaction, fitness_judge, etc.) from the main agent/gateway pool.

Purpose: Eliminate the v11.0 smoke-test §3.1 failure mode where a gateway burst exhausts all 4 GLM keys, leaving the round table with no usable credentials halfway through a 9-panelist session. Auxiliary tasks get their own pool (separate env vars `GLM_AUX_API_KEY_1..4`, with fallback to `GLM_API_KEY` for single-key operators), so gateway traffic cannot starve round-table calls.

Output:
- `agent/credential_pool.py` extended with named-pool helpers (`load_aux_pool`, `load_named_pool`, `_seed_aux_env`).
- `tests/agent/test_credential_pool_aux_isolation.py` (NEW) — TDD coverage for pool isolation + env var fallback + no-breaking-change.

- `agent/glm_throttle.py` modified — `acquire_slot` / `try_acquire_slot` accept optional `pool_name` kwarg (backward-compatible).
- `cli-config.yaml.example` modified — `GLM_AUX_API_KEY_1..4` documentation block.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env vars → process | `GLM_AUX_API_KEY_1..4` + `GLM_API_KEY` read from `os.environ` + `~/.hermes/.env` |
| auth.json → process | `read_credential_pool("auxiliary:zai")` reads persisted entries from disk |
| process → auth.json | `write_credential_pool("auxiliary:zai", ...)` persists seeded entries back to disk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-59-01-S | Spoofing | env-var seeding (`_seed_aux_env`) | mitigate | Match env-var source labels with high specificity (`env:GLM_AUX_API_KEY_1`, NOT a generic `env:aux` label) so audit trails distinguish dedicated aux keys from `GLM_API_KEY` fallback. Test 3 verifies this label discipline. |
| T-59-02-T | Tampering | auth.json named-pool storage key | mitigate | Named-pool entries persist under `credential_pool["auxiliary:zai"]`, separate from default `["zai"]`. Test 5 verifies the storage-key isolation, preventing cross-pool writes. |
| T-59-03-I | Information disclosure | `logger.debug` lines in pool selection | accept | Existing pool-selection logs at DEBUG level expose label + id (NOT raw tokens), matching Phase 52-58 pattern. No new disclosure surface introduced. |
