<objective>
Build the memory compaction pass — when an agent's memory store exceeds `max_records=500`, archive the oldest archival-tier records and summarize the working-tier into a core-tier summary record (per PITFALLS §P9 mitigation 1+2+3). Post-compaction state must satisfy the 3-tier structure (core ≤10 / working ≤100 / archival ≤10000), with originals archived (not deleted) and the summary record carrying `source_record_ids` chain for provenance.

Purpose: P9 mitigation (memory size growth). Without compaction, memory grows unbounded → context overflow + summarization detail loss. This is the load-bearing gate for memory tiering at scale (per `05-POC-PLAN.md` §4.4).

Output: `agent/memory_compaction.py` (NEW module, ~120 lines) + 600-record synthetic fixture + tests/v11-compaction/ test suite.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mem0 backend → compaction | Compaction reads + writes mem0 records. A poisoned mem0 store (corrupted ConfidenceInferenceConflict or schema-violating record) could propagate to the summary record. |
| GLM dispatch → summary content | LLM-generated summary record is inserted into mem0 as active core-tier. A hallucinated summary would dominate future retrievals. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-55-01 | Tampering | mem0 backend reads | accept | mem0 is operator-controlled local backend; not adversarial. Compaction is read-then-write, no untrusted input crosses this boundary. |
| T-55-02 | Information Disclosure | summary record content | mitigate | GLM summary prompt explicitly instructs "preserve all distinct facts, do not omit." Summary `confidence` = mean of source confidences (no inflation). Bias canary (Phase 54) gates post-compaction core-tier records on next bias canary run. |
| T-55-03 | Denial of Service | unbounded compaction loop | mitigate | `compact_memory` checks `record_count <= DEFAULT_MAX_RECORDS` and returns `triggered=False` without dispatch. Idempotent — calling twice on the same store produces the same report (already compacted). |
| T-55-04 | Elevation of Privilege | dry-run bypass | mitigate | `dry_run: bool = True` default (EVAL-06 hard constraint). AST-walk in 55-03 verifies no caller invokes `compact_memory(dry_run=False)` outside the approved entry points. |
| T-55-SC | Tampering | no new packages installed | accept | This plan installs zero packages (planning_context constraint #7 "No new packages"). |
</threat_model>
