       AST parent tracking pattern (cite for executor):
       ```python
       import ast
       source = open("agent/curator.py", encoding="utf-8").read()
       tree = ast.parse(source)
       parent_map = {}
       for node in ast.walk(tree):
           for child in ast.iter_child_nodes(node):
               parent_map[id(child)] = node

       def is_inside_not_dry_run_guard(call_node):
           node = call_node
           while node in parent_map:
               parent = parent_map[node]
               if isinstance(parent, ast.If):
                   test = parent.test
                   if (isinstance(test, ast.UnaryOp) and isinstance(test.op, ast.Not)
                           and isinstance(test.operand, ast.Name) and test.operand.id == "dry_run"):
                       return True
               node = parent
           return False
       ```

       The AST-walk is a STATIC analysis — it does not execute any code. It only verifies source structure. This is the same approach v6.0 used (cite `tests/test_curator_audit.py::TestNonBypassableHumanInLoop`).
