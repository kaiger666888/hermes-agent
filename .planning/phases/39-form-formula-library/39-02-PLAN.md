<!-- Formula JSON schema — every file must conform. Plan 01 defines the Pydantic model; this is the JSON shape it validates. -->

Each formula JSON file has this exact shape (matches Plan 01 `Formula` Pydantic model):

```json
{
  "formula_id": "urban-fantasy-light-01",
  "genre": "都市奇幻",
  "mood": "轻喜剧",
  "pacing": "fast-cut",
  "hook_pattern": "contrast",
  "characters": ["hidden-boss", "everyman-sidekick"],
  "runtime_sec": 90,
  "platform_fit": [
    {"platform": "抖音", "fit_score": 0.9},
    {"platform": "快手", "fit_score": 0.8},
    {"platform": "B站", "fit_score": 0.6}
  ],
  "citation": {
    "source": "Notion 创作方向 page 32811082-af8e-8009-b097-d19a5027b46f §核心 DNA",
    "source_type": "notion",
    "fair_use_status": "verbatim-spec",
    "verified_date": "2026-06-26"
  },
  "verified_date": "2026-06-26",
  "eval_score": null
}
```

**Field constraints (enforced by Plan 01 schema):**
- `formula_id`: unique kebab-case string; pattern `{genre_slug}-{mood_slug}-01` (e.g. `urban-fantasy-light-01`). Genre slug mapping: 都市奇幻→`urban-fantasy`, 悬疑反转→`mystery-twist`, 家庭情感→`family-emotion`, 校园青春→`campus-youth`, 职场商战→`workplace`. Mood slug: 轻喜剧→`light`, 虐心→`angst`.
- `genre`: one of `["都市奇幻","悬疑反转","家庭情感","校园青春","职场商战"]`
- `mood`: one of `["轻喜剧","虐心"]`
- `pacing`: short string descriptor (e.g. `fast-cut`, `slow-burn`, `mid-tempo`)
- `hook_pattern`: one of 5 types from `hook_retention/references/three-second-hooks.md`: `emotional` / `suspense` / `conflict` / `contrast` / `emotional_peak`
- `characters`: list of 2-5 archetype slugs (kebab-case); vary across formulas (e.g. `hidden-boss`, `everyman-sidekick`, `foiled-rival`, `tragic-mentor`, `prodigal-returnee`, `underdog-rookie`, `matriarch-iron`, `simp-loyal`, `ex-convict-reborn`, `naive-newcomer`)
- `runtime_sec`: int in [60, 600]
- `platform_fit`: list of 2-6 entries; platforms from `["抖音","快手","B站","小红书","视频号","红果"]`; fit_score float in [0.0, 1.0]
- `citation.source`: string identifying exact source (Notion page+section / book title+page / benchmark ID)
- `citation.source_type`: `notion` / `public-book` / `kais-benchmark`
- `citation.fair_use_status`: `verbatim-spec` (reproduced strategy spec) / `paraphrased` (rewritten in our words) / `derived-analysis` (original interpretation based on source)
- `citation.verified_date` + top-level `verified_date`: ISO date (YYYY-MM-DD); use `2026-06-26` for all seed formulas
- `eval_score`: `null` for all 10 seed formulas (v9.0 ships no eval data; v6.0 eval gate 回填 is post-DATA phase)

**Filename convention:** `formula_{genre_slug_underscore}_{mood_slug}_01.json` where genre_slug uses underscores (e.g. `urban_fantasy`), mood_slug is `light` / `angst`. The `_01` index is uniform since there is exactly one formula per cell.

**Citation source pool (FORM-03):**
- Notion page `32811082-af8e-8009-b097-d19a5027b46f` ("心流♥ → aigc开发 → 创作方向") — primary source; cite with section anchor
- 公开爆款公式书 (public short-drama playbook books, e.g. 《短视频爆款公式》《短剧创作指南》) — cite with book title + page; use `paraphrased` fair-use status
- kais-movie-agent historical benchmark — cite with benchmark ID + run date; use `derived-analysis` fair-use status

**Per CLAUDE.md copyright rule:** every formula MUST have a citation. No uncited formulas. Citations are fair-use only — paraphrased methodology + factual specs, no verbatim script/dialogue beyond short factual phrases already in Notion.
