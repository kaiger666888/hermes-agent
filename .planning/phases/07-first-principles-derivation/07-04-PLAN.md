**§6.1 — Walk Category 1 (first-principles misapplication) — 6 failure modes:**

For each of the 6 failure modes (1.1 through 1.6), write a subsection with this structure:

```markdown
### Audit 1.X — [failure mode name from PITFALLS §1.X]

**Audit prompt:** [the audit prompt from RESEARCH §2 table]

**Verdict:** PASS / FAIL / CONDITIONAL
- If PASS: state briefly why the audit is satisfied
- If FAIL: state what the derivation missed + recommend Phase 8+ remediation
- If CONDITIONAL: state what condition must hold for the audit to pass (e.g., "PASS conditional on §4.1 creative_source derivation being re-verified in Phase 8 with C1-C7 filter")
```

The 6 audits to write (audit prompts from RESEARCH §2):
- **1.1 "First principles" as decorative language** — prompt: does any node's `derivation` field read "every pipeline has this"? Does the derivation section length exceed or match the node-description section? Does the word "obviously" appear anywhere?

- **1.3 Premature optimization for current-gen models (Sora/Kling/Veo)** — prompt: do any candidate node IDs contain model-specific terms? Are node IDs expressed at the user-value layer?

- **1.6 Reverse-engineering the desired answer into "first principles"** — prompt: does every node carry a steelman-the-elimination paragraph? Are alternatives-considered logs present and substantive?
