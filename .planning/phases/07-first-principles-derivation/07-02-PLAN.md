**§3.1 — From Q1 ("what does audience consume?") to intermediate conclusions**

Walk the reasoning in numbered steps (D1.1, D1.2, D1.3, ...) where each step:
- States the claim
- Tags it with epistemic-status (`[physical: ...]` or `[psychological: ...]` etc.)
- Cites the corpus source(s) supporting the claim
- Classifies the underlying assumption as `contingent` or `validated-invariant`
- Notes any inherited assumption being questioned (per ARCHITECTURE §3.3)

Example structure (illustrative — the actual claims are for the executor to derive):
- D1.1: The audience consumes a coherent emotional-cognitive experience, not a video file. `[psychological]` (audience-reception is human-nature contingent but stable; not a platform algorithm). Corpus: Bazin's ontology of cinema (`film-philosophy-bazin-tarkovsky.md`). Assumption: `validated-invariant` (extraordinary evidence would be required to overturn).
- D1.2: A coherent experience requires narrative meaning + perceptual richness + emotional arc as joint, not separate, deliverables. `[psychological]`. Corpus: Tarkovsky's "Sculpting in Time" (cited in `film-philosophy-bazin-tarkovsky.md` and `narrative-revolution-and-modernism.md`). Assumption: `validated-invariant`. **Inherited-assumption-questioned**: kais-movie-agent V1-V8 separated "scenario → storyboard → shots" as sequential phases; D1.2 implies these are JOINT properties of one experience, not separable stages.
- D1.3: Therefore the pipeline must produce integrated experience, not a sequence of asset files passed forward. `[platform-algorithmic]` only if making a platform-specific claim, otherwise `[psychological]`. Corpus: 03-拍摄/cinematographer-masterclass. Assumption: `contingent` (the V1-V8 JSON asset bus is a workflow choice, not a perceptual invariant).

Continue this pattern through ~5-8 derivation steps for Q1, ending with intermediate conclusions about what the pipeline's TOP LEVEL must produce (e.g., "the pipeline root node must produce a coherent experience spec, not a script or a storyboard as separate artifacts").

**§3.2 — From Q2 ("what determines 短剧/微电影 quality?") to intermediate conclusions**

Same structure. Use corpus from §2.2 (Murch Rule of Six + editing-rhythm-pacing + color-grading-strategy). End with intermediate conclusions like "the pipeline must own cross-node coherence as a first-class value driver (not a side-effect of good per-node specs)" per PITFALLS 5.2.

**§3.3 — From Q3 ("what can AI actually accelerate?") to intermediate conclusions**

Same structure. Use corpus from §2.3 (post-production procedural tasks + animation + budget allocation + kais V8 architecture + STACK §5 LLM-story-gen refs).

For Q3, classify each "acceleratable" operation by current `tool-capability` volatility: e.g., "AI accelerates color grading assist (tool-capability: stable_2026 — current models are good)" vs "AI accelerates multi-shot identity lock (tool-capability: evolving — current LoRA works but native cross-shot identity is research_bet)".

End with intermediate conclusions like "the pipeline must distinguish AI-accelerated operations (no human review needed) from AI-bounded operations (human-in-loop required) from AI-native operations (no traditional precedent)".

**§3.4 — From Q4 ("what can AI never replace?") to intermediate conclusions**

Same structure. Use corpus from §2.4 (film-philosophy-bazin-tarkovsky + screenwriting + acting). End with intermediate conclusions like "the pipeline must include a creative-source node that mines story kernels from lived experience, not from training data" and "the pipeline must include a consultative theory_critic that is invoked manually (META-06), not auto-triggered".

For Q4, carefully preserve validated-invariants (Murch, Bazin, Stanislavski) WITHOUT discarding them as "bias" per PITFALLS 1.2.

**§3.5 — Synthesis: from intermediate conclusions to "shape of the candidate set"**

End §3 with a synthesis paragraph: the four intermediate-conclusion sets (from Q1-Q4) jointly imply the candidate node set must have certain structural properties:
- A root node producing integrated experience spec (from Q1)
- A coherence-owner node / sub-step (from Q2)
- AI-acceleration nodes distinguished by tool-capability class (from Q3)
- AI-bounded nodes for creative intent + consultative theory_critic (from Q4)

The synthesis should NOT yet enumerate the final node IDs — that's Plan 03's job. §3 ends with "the candidate set must look roughly like [structural sketch]; §4 enumerates the specific nodes with full per-node rigor".
