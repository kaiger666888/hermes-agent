## Phase Boundary

Phase 1 builds the `compliance_marketing` expert end-to-end — the legal gate that ensures content generated by the Movie-Experts Suite can be lawfully distributed in China under the 2025-09-01 网信办 AI 标识办法 and the 2026-04-01 AI 漫剧 备案 regime. This expert also embeds per-platform distribution rules (抖音 / 快手 / 微信小程序剧) and a 爆款 catalog that cross-references 审核风险 with 降级方案.

In scope:
- Author `skills/movie-experts/compliance_marketing/SKILL.md` (bilingual EN YAML + CN prose)
- Author 5 reference files:
  - `references/cn-content-rules.md` (AI 标识办法 + 备案 regime + 8-category 红线 checklist)
  - `references/platform-specs-douyin.md`
  - `references/platform-specs-kuaishou.md`
  - `references/platform-specs-miniprogram.md`
  - `references/viral-element-catalog.md` (5-type taxonomy + risk badges + 降级方案)
- Publish `_shared/platform-comparison.md` cross-platform matrix
- Author 5 eval prompts covering diverse compliance scenarios
- Wire bidirectional `related_skills` edges to screenplay, editor, hook_retention, style_genome, drawer
- Add `compliance_marketing` to scanner allowlist via `known-external-models.yaml` (if needed)

Out of scope:
- Implementation of AIGC labeling code (skill only references the workflow; no Python/JS)
- Legal analysis beyond what's needed for content generation compliance
- Automated platform-guideline scraper (manual quarterly refresh only)
- Phase 2 HOOK expert (but COMPLI must leave a stable contract for HOOK's paywall compliance constraints)
- Phase 6 full N≥20 eval run (Phase 1 only ships the prompts)

## Implementation Decisions

### CN Legal Framework Scope
- **Covered regulations:** 网信办 AI 标识办法 (2025-09-01) + AI 漫剧 备案 regime (2026-04-01) + 8-category 内容审核红线 checklist
- **8 红线 categories:** 政治敏感 / 暴力血腥 / 色情低俗 / 未成年人保护 / 民族宗教 / 歧视侮辱 / 虚假宣传 / 版权侵权 (covers 网信办 + 广电总局 + 市场监管总局)
- **AIGC labeling workflow:** Full coverage — visible watermark + metadata tag + content disclosure script-line template (required for legal distribution after 2025-09-01)
- **AI 漫剧 备案 trigger matrix:** Document episode count / total runtime / commercial intent / payment mechanism thresholds that trigger 备案 requirement

### Platform Spec Depth & Structure
- **Same depth per platform** (抖音 / 快手 / 微信小程序剧) — each ref ≥ 200 lines, equal weight
- **Identical H2 sections across all 3 platform refs:** `## 分发规则` / `## 内容红线` / `## 付费机制` / `## 备案要求` / `## 爆款公式`
- **Drift handling:** Each ref carries `verified_date: 2026-06` + `## Refresh Cadence` (quarterly check) + `## Drift Signals` (events that warrant off-cycle re-verification, e.g., new 网信办 notice, platform policy update announcement)
- **Cross-platform comparison:** `_shared/platform-comparison.md` matrix (rows: platforms; cols: 付费门槛 / 红线差异 / 备案触发 / 推荐时长)

### 爆款 Catalog vs 审核风险 Cross-Reference
- **5-type taxonomy:** 情感钩 / 冲突钩 / 反差钩 / 题材钩 / 角色钩
- **Per-catalog-entry schema:** `元素 / 爆款机制 / 审核风险等级 (🟢/🟡/🔴) / 关联红线 §X.Y / 降级方案 (preserves hook strength while reducing risk)`
- **3-tier risk levels:** 🟢 safe / 🟡 context-dependent (review needed) / 🔴 always-rejected
- **8 红线 categories:** Same as legal framework scope above

### Eval Prompts & SKILL.md Body Architecture
- **5 eval scenarios:** (1) AI 标识 + AIGC disclosure for a 短剧 episode (2) 备案 material compilation for 漫剧 (3) Platform cut/edits for 抖音 vs 快手 vs 小程序剧 (4) 红线 review of a borderline-violent scene (5) 未成年人保护 check on a 校园 setup
- **SKILL.md body:** Core capabilities + RAG invocation pattern + per-platform branching with `### 抖音 / ### 快手 / ### 小程序剧` subsections each containing platform-specific 爆款公式 + 红线 highlights
- **Bilingual format:** EN YAML frontmatter + EN H2 headers + CN prose body with concrete examples (matches PROJECT.md decision)
- **related_skills edges:** Bidirectional to/from screenplay (red-line check before finalize), editor (cut compliance), hook_retention (risk-aware hooks), style_genome (visual risk), drawer (poster compliance) — total 5 edges × 2 directions

### Claude's Discretion
- Specific examples and sample dialogues within each ref
- Exact H3 subsection breakdown under each H2 in platform specs
- Order of refs in `## References` table
- Concrete numeric thresholds for 备案 triggers (best-effort from public 广电总局 notices; mark uncertain values with `*estimated*`)
- Final wording of judge prompts in eval files (shape is fixed by Phase 0 judge_prompt.md template)

<specifics>
## Specific Ideas

- **AI 标识办法 core workflow ref must include:**
  - 显式标识 (visible mark) — size, position, duration rules for video
  - 隐式标识 (metadata) — required C2PA-style metadata fields
  - 文本披露 (script-line template) — example disclosure line for dialogue
  - 触发条件 — which content types require labeling (all AIGC commercial distribution)
- **AI 漫剧 备案 matrix should be a Markdown table:** Rows = content categories (e.g., 拟人化动画 / 真人+AIG / 全AIG); Cols = 备案 trigger thresholds (集数 / 总时长 / 商业意图 / 付费机制)
- **Platform specs should each include concrete numeric thresholds:** 抖音 付费门槛 (e.g., 解锁集数 / 单集价格区间); 快手 创作者激励 rules; 小程序剧 备案/付费 dual requirement
- **爆款 catalog should include ≥ 5 entries per taxonomy type (25+ total)** — enough variety for downstream agents to combine
- **降级方案 should preserve ≥ 70% of original hook strength** — heuristic to keep commercial value
- **Eval prompts should exercise both clean cases (easy) and borderline cases (hard)** — at least 2 of 5 prompts must involve 1+ 🟡 risk-level element

</specifics>
