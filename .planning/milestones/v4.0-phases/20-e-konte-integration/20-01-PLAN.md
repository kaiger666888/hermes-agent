## File-level Plan

### F1: NEW `skills/movie-experts/cinematographer/references/e-konte-format.md`

**Target length:** 280-380 行(对齐 cinematographer 现有 refs 密度,axis-rules.md 186 行 / camera-motion-catalog.md 286 行 / shot-grammar.md 175 行;e-konte 加对比表 + 5 层 + 2 案例应达 ~300 行)。

**Structure:**
1. Header + Source + Copyright (Fair Use paraphrase + 日本动画工业体系 + LICENSE.md 引用,对齐 shot-grammar.md §1-7 行模式)
2. Summary — E-Konte 为什么填补 storyboard_designer deprecated 后的东方分镜真空
3. ## E-Konte 5 Annotation Layers — 5 层详解(场景布局 / 镜头角度与运动 / 角色位置表情动作 / 对白音效 / 时间帧数)
   - 每层:CN/EN 双语术语 + 字段 schema + 短剧 9:16 适配 + 1-2 行实例
4. ## 短剧 9:16 E-Konte 修正 — 竖屏 vs 日本动画传统横屏 16:9 的差异
5. ## E-Konte vs Western Storyboard 对比表 — load-bearing 核心表
   - E-Konte 5 层 ↔ Mascelli 8-level shot scale(映射 + 不映射部分)
   - E-Konte Layer 2 ↔ cinematographer 现有 12 camera moves(camera-motion-catalog.md)
   - E-Konte Layer 5 时间帧数 ↔ Western "每页 ≈ 1 分钟"启发式
   - E-Konte ト書き ↔ Western scene description
   - 互补声明:东方分镜语法 vs 西方轴线传统,互补不替代
6. ## 今敏《红辣椒》案例 — 参考引述(1-2 段,不展开 imitate 级细节)
   - 1.5 年分镜周期的精度上限,作为 "E-Konte 可达到的精度上限" 参考
   - 本 ref **不要求** AI 输出达到今敏级精度
7. ## 宫崎骏吉卜力实践 — 参考引述(1-2 段)
   - 宫崎骏亲手绘制分镜 + Layout 制度
   - Layout 与 E-Konte Layer 1+2 的耦合
8. ## Trigger Conditions — 何时选 E-Konte vs Western storyboard
   - 导演风格 = 东方(日本动画 / 中国动画 / 韩国动画 / 学习东方传统的国漫)
   - scene.visual_density ≥ threshold(信息密度高 → 5 层标注收益大)
   - 短剧 60-180s 单集时,默认走 Western storyboard(E-Konte overhead 太高);180s+ 时触发 E-Konte 评估
9. ## 与现有 Mascelli / Arijon / 180°/30° 规则的集成 — 双输出选项
   - cinematographer 输出可选 `shot_intent.json`(西方)+ `e_konte.json`(东方)双 artifact
   - 下游 visual_executor 根据 `e_konte.json` 是否存在选择消费路径
10. ## Output Schema (`e_konte.json`) — 单 shot 5 层标注的 JSON schema
11. ## Anti-Patterns / What NOT to do
12. ## License — Fair Use paraphrase 声明 + LICENSE.md 引用

**License status:** Fair Use paraphrase(对齐 shot-grammar.md / snowflake-method.md 模式),日本动画工业体系出处标注(公开访谈 / 纪录片 / 工业标准术语,不复制具体分镜页或受版权保护的 cut)。
