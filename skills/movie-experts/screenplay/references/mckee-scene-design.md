## Summary

本 ref 把 Robert McKee 在 *Story* (1997) 中提出的四个核心场景设计理论 —— **gap between expectation and result**(期望与结果的鸿沟)、**scene-value-shift**(场景价值转换)、**beat decomposition**(节拍分解)、**turning point vs plot point**(转折点 vs 情节点)—— 适配到 短剧 / 微电影 的单集场景设计。本 ref 是 McKee 场景数值的**唯一真相源**(value-shift rate ≥ 1 per scene / beat count 3-5 per 90s scene / plot point 位置 ~25% & ~75% runtime)—— SKILL.md body 仅引用数字 + 跨链,不重述原理(Phase 1 [CR-01](../../../../../.planning/phases/02-expert-hook-commercial-engine/02-CONTEXT.md) 教训)。术语定义见 [`../../_shared/glossary.md`](../../_shared/glossary.md)。

McKee 的理论之所以对 短剧 尤其重要,是因为 短剧 的单场景 runtime 极短(通常 5-15s),每一秒都必须有戏剧密度。McKee 的 value-shift 规则("每个场景必须至少转换一个价值")是检验"这个场景是否有存在必要"的硬标尺 —— 没有价值转换的场景就是 filler,应被删除或合并。

## The Gap Definition

McKee 对"戏剧"最精确的定义:**戏剧发生在 protagonist(主角)采取行动后,世界的反应与主角的期望之间产生 gap(鸿沟)的那一刻。** 没有 gap,就没有戏剧 —— 只有信息传递。

### Gap 的 4 步解析协议

McKee 给出了 gap 产生的精确 4 步协议(这是 heuristic,不是模糊描述):

1. **主角带着期望采取行动。** 主角做某件事,因为他期待某个结果。期望必须是具体的(不是"我希望事情变好",而是"我递出名片,期待对方尊重我")。
2. **世界做出反应。** 世界(其他角色 / 环境 / 事件)对主角的行动做出反应。反应可以是言语、行为、或环境的改变。
3. **gap 张开。** 反应与期望不一致 —— 反应比期望更糟(主角被羞辱)或更好(主角意外获得帮助)。**关键是反应与期望之间存在质性的断裂,而非程度上的差异。**
4. **主角适应。** 主角基于新的现实调整行动 —— 这个适应行动本身又带着新的期望,可能再次产生 gap(场景深化)或达成结果(场景结束)。

### 2 个 CN 短剧 场景的 gap 解析

**示例 1:男频 装穷打脸(90s 短剧 S1E03 场景)**

- 步骤 1(期望):主角穿着破西装走进前妻的订婚宴,递上礼物(一束便宜的花),**期待**前妻尴尬但收下。
- 步骤 2(反应):前妻的新未婚夫当众把花扔到地上,踩了一脚,说"这种垃圾也配送给我未婚妻?"
- 步骤 3(gap 张开):反应与期望完全相反 —— 主角期待的是"尴尬的礼貌",得到的是"公开羞辱"。gap 极深。
- 步骤 4(适应):主角默默弯腰捡起花,抬头时眼神变了(特写)—— 他决定不再隐藏身份。**这个适应行动带着新的期望(我要复仇),为后续场景的 gap 铺垫。**

**示例 2:女频 替身的爱(120s 短剧 S1E05 场景)**

- 步骤 1(期望):女主(替姐姐代嫁)在洞房夜鼓起勇气向男主坦白"我不是她",**期待**男主愤怒但给她说清楚的机会。
- 步骤 2(反应):男主沉默良久,然后说"我知道"。他从抽屉拿出一封信 —— 是姐姐婚前写给他的,信里说她不会来结婚。
- 步骤 3(gap 张开):反应与期望不同方向 —— 主角期待的是"愤怒 + 质问",得到的是"平静 + 早已知情"。gap 是认知性的:主角以为自己在欺骗,其实从一开始就被看穿。
- 步骤 4(适应):女主愣住,然后问"那你为什么还娶我?"—— 这个问题带着新的期望(期待一个情感答案),为下一场景的 gap 铺垫。

### Gap 深度的 3 级标尺(heuristic)

McKee 没有给 gap 量化,但基于他的案例,可以归纳出 gap 深度的 3 级标尺(这是本 ref 的衍生 heuristic,非 McKee 原文):

| 级别 | gap 描述 | 观众反应 | 短剧 适用性 |
|------|----------|----------|-------------|
| **浅 gap** | 反应与期望有差异,但方向一致(主角期待尊重,得到的是冷漠的尊重) | 观众感受到"微妙的不对劲",但情绪不强烈 | 适合 小程序剧 长集数的内省场景 |
| **中 gap** | 反应与期望方向相反(主角期待尊重,得到的是羞辱) | 观众感受到"冲突",情绪开始波动 | 适合 90s/120s 短剧 的常规场景 |
| **深 gap** | 反应与期望方向相反 + 主角的认知框架被颠覆(主角以为自己骗了对方,其实从一开始就被看穿) | 观众感受到"震撼",情绪峰值 | 适合 [爽点](../../_shared/glossary.md#爽点-satisfaction-beat) / [击中点](../../_shared/glossary.md#击中点-emotional-impact-point) 场景 |

**关键 heuristic:** 90s 短剧 的每个场景至少需要 1 个中 gap 或深 gap;只有浅 gap 的场景应被合并到相邻场景。这与下方 §Value-Shift Rule 的"≥ 1 value-shift per scene"是同一规则的两个表述(gap 是机制,value-shift 是结果)。

### Gap 与 Snyder Beat 的映射(heuristic)

McKee 的 gap 理论与 Snyder 的 beat 模型([`save-the-cat-beat-sheet.md`](./save-the-cat-beat-sheet.md) §The 15 Beats)有精确映射关系:

| Snyder Beat | 典型 Gap 类型 | Gap 深度 | 说明 |
|-------------|---------------|----------|------|
| **Catalyst** | 外部强加的 gap | 深 | 触发事件本身就是主角无法控制的 gap |
| **Debate** | 内在 gap(主角自我质疑) | 中 | 主角对 Catalyst 的反应产生内在冲突 |
| **Break into Two** | 决策 gap | 中 | 主角选择进入新世界,期望与现实可能产生 gap |
| **Fun & Games** | 连续小 gap | 浅-中 | 兑现段的每个 击中点 都是一个小 gap |
| **Midpoint** | 极性反转 gap | 深 | 假胜利 / 假失败是最深的 gap 类型 |
| **All Is Lost** | 最深 gap | 极深 | 主角失去之前获得的一切 |
| **Break into Three** | 觉醒 gap | 中 | 主角从 Dark Night 中觉醒,找到新方案 |
| **Finale** | 兑现 gap | 中-深 | 主角执行新方案,gap 从"期望失败"到"实际成功" |

**关键 heuristic(衍生):** 短剧 创作者可以用这张映射表诊断剧本的"gap 密度"。如果某个 Snyder beat 没有 gap(例如 Catalyst 只是"主角收到通知",没有期望-反应-鸿沟),这个 beat 的戏剧密度不足,需要重写。

## Value-Shift Rule

McKee 的核心场景规则:**每一个场景必须至少转换一个价值 (value) 从正面到负面,或从负面到正面。** 没有 value-shift 的场景就是 filler —— 它没有改变任何戏剧状态,观众看了等于没看。

### 价值 (Value) 的定义

McKee 的"value"是指人类经验中**二元对立的基本维度**。一个场景可以转换的价值包括(但不限于):

| 价值对 | 正面 | 负面 | 短剧 典型应用 |
|--------|------|------|---------------|
| 生 / 死 | 生存 | 死亡 | 战神归来 / 复仇 |
| 爱 / 恨 | 相爱 | 相恨 / 冷漠 | 女频 豪门虐恋 |
| 自由 / 囚禁 | 自由 | 囚禁(物理 / 心理) | 重生复仇 / 替身 |
| 希望 / 绝望 | 充满希望 | 彻底绝望 | 萌宝 / 逆袭 |
| 真相 / 谎言 | 真相揭露 | 谎言维持 | 家族秘辛 / 悬疑 |
| 富 / 贫 | 富有 | 贫穷 | 装穷打脸 / 赘婿 |
| 强 / 弱 | 强大(物理 / 社会地位) | 弱小 | 战神 / 修仙 |
| 荣誉 / 耻辱 | 荣誉 | 耻辱 | 男频 打脸 |

### Value-Shift 的计量规则(heuristic)

**关键 heuristic 1(硬规则):** **每个场景必须至少转换 1 个价值。** 这是 McKee 的零容忍规则。没有 value-shift 的场景应被删除或合并。验证方法:看完这个场景后,问自己"主角的状态(在任一价值维度上)是否与场景开始时不同?" —— 如果答案是"没有变化",这个场景不合格。

**关键 heuristic 2(质量规则):** value-shift 的**幅度**(从 +3 到 -3,或从 -2 到 +2)比方向更重要。一个从"微弱正面"到"微弱负面"的 shift(幅度 1)远不如一个从"强烈正面"到"强烈负面"的 shift(幅度 6)。短剧 的 runtime 紧张,**应优先安排大幅度的 value-shift**,而不是多个小幅度的。

**关键 heuristic 3(密度规则):** value-shift rate = **场景数 / 总 value-shift 次数**。健康指标是 **≥ 1.0**(每个场景至少 1 次 shift)。优秀指标是 **≥ 1.5**(部分场景有 2 次 shift —— 先正后负,或先负后正,形成场景内的"微型弧线")。低于 1.0 意味着有 filler 场景存在。

### 5 个 canonical value-pair 示例(短剧 应用)

1. **爱 / 恨(女频 豪门虐恋):** 场景开始时女主爱男主(正面);场景中男主说出伤人的话,女主的爱转为恨(负面)。value-shift:爱 → 恨。
2. **富 / 贫(男频 装穷打脸):** 场景开始时主角被当作穷人(负面);场景中主角的真实身份被揭露(正面)。value-shift:贫 → 富。
3. **自由 / 囚禁(重生复仇):** 场景开始时主角被冤枉入狱(负面);场景中主角越狱成功(正面)。value-shift:囚禁 → 自由。
4. **真相 / 谎言(家族秘辛):** 场景开始时主角相信养父是好人(谎言 = 负面);场景中主角发现养父是凶手(真相 = 正面,但内容是痛苦的)。value-shift:谎言 → 真相。
5. **强 / 弱(战神归来):** 场景开始时主角被保安拦住(弱 = 负面);场景中主角一个电话调来车队(强 = 正面)。value-shift:弱 → 强。

**关键 heuristic 4:** value-shift 的方向(正→负 vs 负→正)决定了场景的情绪色彩。**正→负的 shift 制造痛苦 / 紧张(适合 All Is Lost 段);负→正的 shift 制造释放 / 满足(适合 Finale / [爽点](../../_shared/glossary.md#爽点-satisfaction-beat) 段)。** 90s 短剧 的典型情绪曲线是:前半段正→负(主角陷入困境),后半段负→正(主角逆袭兑现)。

### Value-Shift 失败模式诊断(heuristic)

以下是 短剧 创作者常犯的 value-shift 失败模式:

| 失败模式 | 描述 | 典型症状 | 修复 |
|----------|------|----------|------|
| **"无 shift 场景"(filler)** | 场景没有任何价值转换 | 删掉这个场景后剧情不受影响 | 删除或合并到相邻场景 |
| **"伪 shift"** | 价值转换了但幅度太小(从 +1 到 -1) | 观众感受不到情绪变化 | 增大幅度(从 +1 到 -3)或换一个价值对 |
| **"shift 方向错误"** | 在需要正→负的段(Fun & Games)写了负→正 | 节奏感混乱(该紧张时反而轻松) | 调整 shift 方向匹配 Snyder beat 的情绪色彩 |
| **"shift 过密"** | 一个 5s 场景里有 3+ 个 value-shift | 观众来不及消化每个 shift | 减少到 1-2 个 shift;把多余的移到相邻场景 |
| **"shift 无因果"** | 价值转换了但没有因果链(为什么突然变了?) | 观众觉得"剧情跳跃" | 用 gap 机制(期望-反应-鸿沟)建立因果链 |

### Value-Shift 的跨场景累积效应(heuristic)

**关键 heuristic 5(衍生):** 单个场景的 value-shift 效果是有限的 —— 真正的情绪冲击来自**跨场景的累积**。一个 90s 短剧 如果有 5 个场景,每个场景 1 个 value-shift,总 shift 次数 = 5。但如果这 5 个 shift 的方向一致(全部负→正,或全部正→负),累积效应远大于 5 个随机方向的 shift。

- **累积升级模式(推荐):** 每个场景的 value-shift 幅度递增 —— 场景 1(shift 幅度 2)→ 场景 2(幅度 3)→ 场景 3(幅度 4)→ 场景 4(幅度 5,All Is Lost 最低点)→ 场景 5(幅度 6,Finale 最高点)。这种递增模式制造"越看越紧张 / 越看越爽"的累积感。
- **波浪模式(备选):** shift 方向交替(正→负 → 负→正 → 正→负 → ...)。这种模式制造"起伏感",适合 悬疑 / 家族秘辛 题材(观众不知道下一个 shift 会向哪个方向)。

## Beat Decomposition

McKee 的 "beat"(节拍)是戏剧结构的**最小单位**:一个 beat = 一次行动-反应的交换。一个场景由多个 beat 组成。McKee 的 beat 不是音乐节拍,而是**行为交换的节拍** —— 角色A做某事,角色B反应,这是一个 beat;角色B的反应引发角色A的新行动,这是下一个 beat。

### Beat 计量规则(heuristic)

**关键 heuristic 1:** **一个典型的 90s 短剧 场景(通常 5-15s runtime)包含 3-5 个 beat。** 低于 3 个 beat 的场景通常是"静态对话"(talking heads),缺乏戏剧密度;高于 5 个 beat 的场景在 短剧 runtime 内会显得仓促(每个 beat < 3s,观众来不及消化)。

**关键 heuristic 2:** beat 内的**情绪强度变化**是 beat 质量的标尺。一个高质量的 beat 中,角色的情绪强度(1-10,见 [`../hook_retention/references/conflict-escalation.md`](../hook_retention/references/conflict-escalation.md) §情绪强度 1-10 的含义)会变化 ≥ 1 级。如果连续 3 个 beat 的情绪强度都是 6/10,这个场景是"平的",应被压缩或删除。

**关键 heuristic 3:** beat 之间的**升级 (escalation)** 是场景张力的来源。理想的 beat 序列是升级式的:beat 1 (情绪 5) → beat 2 (情绪 6) → beat 3 (情绪 7) → beat 4 (情绪 8) → beat 5 (情绪 9,场景高潮)。降级式的 beat 序列(beat 1 情绪 8 → beat 2 情绪 6)只适合场景的"收尾 beat"(将情绪从高潮拉回,过渡到下一场景)。

### 一个完整 90s 场景的 beat 分解示例

**场景设定:** 男频 装穷打脸 短剧 S1E03,场景 runtime 12s(45s-57s,Midpoint 之后 Bad Guys Close In 段)。主角(隐藏身份的富豪)在前妻的订婚宴上被前妻的新未婚夫(反派)当众羞辱。

| Beat # | 行动-反应 | 情绪强度 | 升级 / 降级 | value-shift |
|--------|-----------|----------|-------------|-------------|
| 1 | 反派把主角递的花扔到地上,踩一脚,说"这种垃圾也配?" | 7/10 | (场景起始) | 荣誉 → 耻辱(负) |
| 2 | 主角沉默弯腰捡花,周围宾客窃笑。前妻转过头不看他。 | 8/10 | 升级(+1) | (深化耻辱) |
| 3 | 反派继续嘲讽"听说你现在在工地搬砖?真是落魄。" 主角抬头,眼神变了(特写)。 | 8/10 | 持平(但认知 gap 张开 —— 主角决定不再隐藏) | (内在状态转换:隐忍 → 决意) |
| 4 | 主角掏出手机,拨了一个号码,只说一句"过来"。 | 9/10 | 升级(+1) | 弱 → 强(正,但尚未兑现) |
| 5 | 30 秒后(场景末),窗外传来车队声。反派脸色变了。 | 10/10(场景高潮) | 升级(+1) | (value-shift 兑现的悬念 —— 卡点 cliffhanger) |

**beat 计数:** 5 个 beat,符合"90s 短剧 场景 3-5 beat"规则。情绪强度从 7 升到 10,升级式序列。value-shift:荣誉→耻辱(beat 1-2)→ 弱→强(beat 4-5)。场景合格。

### Beat 分解的题材变体(heuristic)

不同题材的 beat 分解有不同的节奏特征:

| 题材 | 典型 beat 数(90s 场景) | beat 间 隔 | 情绪强度范围 | 升级模式 |
|------|------------------------|------------|--------------|----------|
| **[男频](../../_shared/glossary.md#男频-male-oriented-channel) 复仇** | 4-5 | 2-3s/beat | 6-10 | 陡升(最后 2 beat 跳跃式升级) |
| **[女频](../../_shared/glossary.md#女频-female-oriented-channel) 闪婚** | 3-4 | 3-4s/beat | 4-8 | 渐升(每个 beat +1-2) |
| **[女频](../../_shared/glossary.md#女频-female-oriented-channel) 萌宝** | 3-4 | 3-4s/beat | 3-7 | 波浪(有升有降,萌点是降) |
| **小程序剧 悬疑** | 3-5 | 2-4s/beat | 5-9 | 阶梯(每 2 beat 一个台阶) |
| **男频 战神** | 4-5 | 2-3s/beat | 7-10 | 陡升(战力展示是跳跃式升级) |

**关键 heuristic 4(beat 级):** beat 之间的情绪强度升级幅度应 ≥ 1 级(在 1-10 尺度上)。如果连续 2 个 beat 的情绪强度相同(例如都是 7/10),第二个 beat 是"冗余 beat",应被合并到第一个 beat 或删除。

### Beat 的"收尾 beat" 特殊规则(heuristic)

**关键 heuristic 5(beat 级):** 每个场景的**最后一个 beat**(收尾 beat)通常是**降级式**的 —— 它将情绪从场景高潮拉回,为过渡到下一场景做准备。收尾 beat 的情绪强度通常比前一个 beat 低 1-2 级。例外:如果场景的最后一拍是 [卡点](../../_shared/glossary.md#卡点-paywall-cliffhanger-paywall-moment) cliffhanger,则收尾 beat 保持高强度(不降级),作为悬念锚定。

- **降级式收尾 beat 示例:** 场景高潮 beat(情绪 9)→ 收尾 beat(情绪 7,主角离开场景,镜头拉远)→ 过渡到下一场景。
- **卡点 收尾 beat 示例:** 场景高潮 beat(情绪 9)→ 卡点 beat(情绪 10,反派敲门 / 电话响起 / 真相揭露)→ 集末悬念。

### Beat 分解的验证清单

每个 短剧 场景的 beat 分解应通过以下验证:

1. [ ] beat 数在 3-5 之间(90s 场景)/ 5-8 之间(180s 场景)
2. [ ] beat 之间的情绪强度升级 ≥ 1 级(除收尾 beat)
3. [ ] 至少 1 个 beat 产生 gap(期望-反应-鸿沟)
4. [ ] 至少 1 个 beat 产生 value-shift
5. [ ] 收尾 beat 要么降级(过渡),要么保持高强度(卡点)

## Turning Point vs Plot Point

McKee 区分两个常被混淆的概念:**Turning Point(转折点)** 是单个 beat,它张开一个 gap;**Plot Point(情节点)** 是场景级事件,它把故事"旋"进一个新的幕(act)。

### Turning Point(转折点)

- **定义:** 场景内的单个 beat,主角的行动产生 gap,导致主角的认知或行动方向改变。
- **粒度:** beat 级(比场景小)。
- **功能:** 推动场景内的戏剧密度。
- **密度:** 一个场景可以有 1-3 个 turning points(对应 beat 序列中的升级点)。
- **短剧 例子:** 上方 beat 分解示例中,beat 3(主角眼神变了,决定不再隐藏)是一个 turning point —— 它改变了主角的行动方向(从隐忍到反击)。

### Plot Point(情节点)

- **定义:** 场景级事件,它把故事从一个幕旋进下一个幕。Syd Field 称之为 Plot Point I (旋入第二幕) 和 Plot Point II (旋入第三幕);Snyder 的 Catalyst / Break into Two / All Is Lost / Break into Three 都是 plot point(见 [`save-the-cat-beat-sheet.md`](./save-the-cat-beat-sheet.md) §The 15 Beats)。
- **粒度:** 场景级(比 beat 大)。
- **功能:** 推动整体叙事结构的幕转换。
- **密度:** 一个典型的三幕结构有 2 个核心 plot point(Plot Point I 在 ~25% runtime,Plot Point II 在 ~75% runtime)。
- **短剧 位置(heuristic):**
  - **90s 短剧:** Plot Point I 在 ~22s(20-25s),Plot Point II 在 ~67s(64-70s,对应 Snyder All Is Lost)
  - **180s 短剧:** Plot Point I 在 ~45s(40-50s),Plot Point II 在 ~135s(130-140s)
  - **60s 短剧:** Plot Point I 在 ~15s(12-18s),Plot Point II 在 ~45s(42-48s)

### 关键区分(heuristic)

**关键 heuristic:** Turning Point 是"场景内部的齿轮",Plot Point 是"幕之间的铰链"。**一个合格的 plot point 场景通常包含 ≥ 1 个 turning point beat**(否则 plot point 缺乏戏剧密度,只是"事件发生了"但没有"主角如何反应")。反向不成立 —— 一个 turning point 不一定是 plot point(大多数 turning point 只推动场景内部,不旋动幕)。

**短剧 常见错误:** 创作者把 plot point 当成 turning point 来写 —— 只交代"事件发生了"(例如"主角收到解雇通知"),但不写主角的 gap 反应(主角期待什么?世界如何反应?gap 在哪里?)。这导致 plot point 场景缺乏戏剧密度,观众感觉"剧情在推进但没有情绪"。

### Plot Point 场景的诊断清单

每个 plot point 场景(对应 Snyder 的 Catalyst / Midpoint / All Is Lost / Break into Three)应通过以下验证:

1. [ ] 场景中有 ≥ 1 个 turning point beat(主角的 gap 反应)
2. [ ] turning point beat 改变了主角的行动方向(从 A 到 B)
3. [ ] plot point 事件本身是外部强加的(不是主角主动选择的)—— 例外:Break into Three 是主角主动找到解决方案
4. [ ] plot point 后的场景情绪色彩与 Snyder beat 匹配(Catalyst = 中性→紧张;Midpoint = 极性反转;All Is Lost = 最低点;Break into Three = 觉醒)
5. [ ] plot point 的 value-shift 幅度 ≥ 3(比常规场景的 shift 更大)

### Turning Point 与 Gap 的因果关系(heuristic)

**关键 heuristic(衍生):** turning point 的本质是 gap 的"觉察瞬间"。主角在 gap 张开后,可能在 1-3 个 beat 内才"觉察"到 gap(并改变行动方向)。这个"觉察瞬间"就是 turning point。

- **即时 turning point:** gap 张开后,主角在同一 beat 内觉察并改变方向。适合快节奏 短剧(90s 形态)。
- **延迟 turning point:** gap 张开后,主角在 2-3 个 beat 后才觉察。适合慢节奏 小程序剧(180s 形态),允许更深的心理戏。
- **未觉察 turning point(anti-pattern):** gap 张开后,主角始终未觉察,继续按原方向行动。这导致场景缺乏戏剧密度 —— 观众看到了 gap 但主角没有反应,产生"主角怎么这么迟钝"的挫败感。

## Refresh Cadence

- **年度复核:** 重新验证 McKee 的 value-shift rate(≥ 1 per scene)是否仍为场景质量标尺;检查是否有新的场景理论研究修正 beat 计数规则

- **A/B 测试结果触发:** 若 完播率 数据显示"没有 value-shift 的场景"在某些平台 / 题材下 完播率 不降反升,需重新评估 value-shift 规则的普适性
- **新 value-pair 涌现触发:** 若 短剧 行业出现新的核心价值对立(例如"虚拟 / 真实"在元宇宙题材),补充 §Value-Shift Rule 的价值表
- **跨题材 beat 计数漂移触发:** 若不同题材的最佳实践 beat 计数发生系统性变化,更新 §Beat Decomposition 的题材变体表
- **gap 深度标尺失效触发:** 若观众对"浅 gap"的容忍度提升,修订 §The Gap Definition 的 3 级标尺
- **负责模块:** screenplay SKILL.md body 的 narrative_tension metric 定义依赖本 ref

## Drift Signals

以下信号提示本 ref 可能已过时,需要重新验证:

1. **value-shift 规则失效:** 若 A/B 测试显示"纯氛围场景"(无 value-shift)在某些 短剧 题材下(例如治愈系 / ASMR 系) 完播率 更高,需修订 §Value-Shift Rule 的零容忍措辞

3. **gap 深度标尺失效:** 若观众对"浅 gap"的容忍度提升(算法偏好更温和的冲突),需修订 §The Gap Definition 的 3 级标尺
4. **plot point 位置漂移:** 若 Snyder beat budget 与 短剧 实际 plot point 位置出现系统性偏差,需重新校准 §Turning Point vs Plot Point 的 runtime 位置表
