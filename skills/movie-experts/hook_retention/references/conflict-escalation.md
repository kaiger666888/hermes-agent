## The 阶梯式 Escalation Ladder

阶梯式升级 (step-ladder escalation) 是 短剧节奏设计的核心模型。一集典型 90s 短剧应按以下 5 级阶梯逐步推高情绪强度,而不是均匀分布事件。每一级阶梯对应 [CONTEXT D-6](../../../../../.planning/phases/02-expert-hook-commercial-engine/02-CONTEXT.md) marker schema 中的一个 marker 类型(钩子 / 击中点 / 爽点 / 卡点)。

| Rung | 名称 | 触发点 | 情绪强度 (1-10) | 典型位置 | 对应 marker |
|------|------|--------|-----------------|----------|-------------|
| 1 | **钩子锚定** (hook-pin) | 0-3s | 6/10 | opening | 钩子 |
| 2 | **第一次冲击** (first 击中点) | ~10-15s | 7/10 | first beat | 击中点 (软峰) |
| 3 | **中段升级** (mid-escalation) | ~30-45s | 8/10 | mid | 击中点 (硬峰) |
| 4 | **[爽点](../../_shared/glossary.md#爽点-satisfaction-beat)峰值** (爽点 peak) | ~70-80% of episode | 9-10/10 | pre-卡点 | 爽点 |
| 5 | **[卡点](../../_shared/glossary.md#卡点-paywall-cliffhanger-paywall-moment) cliffhanger** | episode end | unresolved (悬而未决) | episode end | 卡点 |

### 情绪强度 (1-10) 的含义

情绪强度是一个**主观代理指标** (subjective proxy),不是实测的生理数据。它的标定方式:

- **1-3/10:** 观众情绪平稳,主要是"理解剧情"的认知负荷,不产生强情绪反应。
- **4-6/10:** 观众开始"被卷入" —— 注意力集中,情绪开始波动(轻微紧张 / 好奇 / 共情)。
- **7-8/10:** 观众有明确的生理反应 —— 心跳加速 / 屏息 / 紧张得坐直身体。
- **9-10/10:** 观众情绪峰值 —— 眼眶湿润 / 拳头握紧 / 不由自主出声("卧槽"/"太爽了")。

强度数字是公开观察的聚合估算(`*estimated*`),实际效果需 A/B 测试验证。

### 阶梯的设计原则

阶梯式模型的核心设计原则:

1. **逐级推高,不允许平级:** 第 N+1 级的情绪强度必须严格高于第 N 级。若 mid-escalation 与 first 击中点 强度持平,观众会感觉"剧情停滞"。
2. **每级之间留出"喘息帧":** 不要把情绪强度从 7/10 直接推到 9/10 而中间没有缓冲。典型缓冲长度 = 3-5s(可以是一个日常细节镜头 / 一个微表情特写)。这避免了情绪疲劳 (emotional fatigue)。
3. **钩子锚定 (Rung 1) 与 爽点峰值 (Rung 4) 必须有因果链:** 第 4 级兑现的爽点必须直接回应第 1 级的钩子承诺。例如:钩子锚定"被陷害" → 爽点峰值"反杀真凶"。若没有因果链,观众会觉得"开场骗我进来,后面是另一部剧"。
4. **卡点 cliffhanger (Rung 5) 是"未解"状态:** 这一集的卡点必须留一个**未解的悬念**,直接驱动观众点开下一集(或在 [小程序剧](../../_shared/glossary.md#小程序剧-mini-program-drama) 中付费解锁)。

## 击中点 Placement Density

[击中点](../../_shared/glossary.md#击中点-emotional-impact-point) (emotional-impact point) 是阶梯式升级的中间台阶 —— 介于钩子与爽点之间的"次级情绪峰"。它的密度直接决定一集 短剧 的"信息密度感"。

### 密度规则(观察值,`*estimated*`)

以下数字是行业公开观察的聚合估算,不是平台实测数据。它们是本 ref 的**唯一真相源** —— [`paywall-design.md`](./paywall-design.md) (02-02) 与 SKILL.md body (02-03) 必须引用这些数字而非重新定义。

| 节奏层级 | 击中点类型 | 间隔 | 单集 90s 短剧典型数量 *estimated* |
|----------|-----------|------|----------------------------------|
| 软峰 | 情绪微冲击(一个表情 / 一句台词 / 一个反应镜头) | **每 10-15 秒** 至少 1 个 | 5-7 个 |
| 硬峰 | 情绪大冲击(关键剧情转折 / 关系破裂 / 身份揭穿) | **每 30-45 秒** 至少 1 个 | 1-2 个 |
| 总计 | 软峰 + 硬峰 | — | **6-9 个 击中点** 总计 |

### 密度的底层逻辑

为什么是这些具体数字?基于以下公开观察的推理:

1. **观众注意力衰减曲线:** 短剧观众的注意力不是线性衰减,而是阶梯式衰减。每 10-15 秒会有一次"注意力低谷",若此时没有击中点 (软峰) 重新拉起,观众的上滑概率显著上升。这一观察来自 MCN 公开创作者访谈与平台官方运营指南的聚合。
2. **算法偏好高密度内容:** 抖音 / 快手 / 微信小程序剧 的推荐算法权重中,完播率与互动率(转发 / 评论 / 点赞)是核心指标。高密度击中点 → 更高的完播率 → 更高的算法分发权重。这是公开的平台运营手册反复强调的规则。
3. **硬峰的"重置效应":** 一个硬峰(30-45s 间隔)会"重置"观众的注意力曲线 —— 让观众从"逐渐疲劳"状态重新进入"高度投入"状态。这就是为什么硬峰间隔不能太短(否则疲劳累积),也不能太长(否则观众在中段流失)。

### 密度的边界(过度 vs 不足)

- **密度过低(< 5 个击中点 / 90s):** 观众感觉"剧情拖沓",完播率显著下降。常见于新创作者的早期作品 —— 误把"铺垫"等同于"剧情"。
- **密度过高(> 10 个击中点 / 90s):** 观众感觉"信息过载 / 情绪疲劳"。每个击中点都被稀释,反而没有一个能真正击穿。常见于试图"塞满爆点"但缺乏节奏感的作品。

## 爽点 Placement Strategy

[爽点](../../_shared/glossary.md#爽点-satisfaction-beat) (satisfaction beat) 是阶梯式升级的最高峰 —— 观众情绪的最强满足瞬间。它的位置与铺垫直接决定一集 短剧 的"是否值得追看"。

### 放置规则(观察值,`*estimated*`)

| 规则 | 数值 *estimated* | 说明 |
|------|------------------|------|
| **位置** | 70-80% of episode runtime | 爽点放在 70-80% 进度处(典型 90s 短剧 = 63-72s 处) |
| **前置铺垫** | ≥ 30s setup | 爽点必须有 ≥ 30s 前置铺垫(callback 到 钩子锚定 或 早先 击中点) |
| **强度** | 9-10/10 | 爽点必须是本集情绪强度最高点 |

### 为什么是 70-80%?

- **太早(< 60%):** 爽点兑现后,剩余 40% 剧情失去动力。观众感觉"高潮已过",完播率断崖式下跌。
- **太晚(> 85%):** 爽点与卡点 cliffhanger 距离太近,观众来不及"消化"爽点就被卡点拉走。这会让爽点的爽感被卡点的悬念感冲淡。
- **70-80% 的甜区:** 爽点兑现后,留 20-30% 的剧情空间用于"收尾"(主角反应 / 关系升华 / 反派的最终反应),然后再进入卡点 cliffhanger。这个节奏让观众既满足于爽点,又因卡点继续追看。

### 爽点必须 SETUP ≥ 30s

爽点不能"凭空出现"。它必须有 ≥ 30s 的前置铺垫 —— 通过一个或多个早先的击中点 / 钩子锚定 暗示"这个爽点即将到来"。铺垫的形式:

- **直接铺垫:** 主角宣告"三天后我会让他们全部跪下" → 三天后兑现(经典男频慕强模式)。
- **间接铺垫:** 通过多个小事件累积主角的"愤怒值",最终在 70% 处爆发(典型复仇爽点)。
- **反向铺垫:** 通过展示反派的"得意"铺垫主角的"反杀即将到来"(对比式铺垫)。

若爽点无前置铺垫,观众会觉得"强行塞糖",反而降低满足感。

### 爽点 Strength Tiers

爽点本身也有强度分级。这一分级**借用** [`three-second-hooks.md`](./three-second-hooks.md) 的 5 级符号体系,但语义针对"回报"而非"钩子":

| 符号 | 名称 | 爽点表现 *estimated* |
|------|------|---------------------|
| 🎯 | **cathartic** (宣泄型) | 爽点强度 10/10;观众生理反应明显(出声 / 拳头握紧 / 截图分享)。典型:复仇终于完成 / 真相揭穿 / 打脸反派。 |
| ✅ | **satisfying** (满足型) | 爽点强度 8-9/10;观众情绪得到明确回报,但未达到宣泄程度。典型:阶段性小胜 / 一次反击 / 一个揭穿。 |
| ⚠️ | **muted** (弱化型) | 爽点强度 5-7/10;爽点存在但被稀释(铺垫不足 / 兑现时机过早 / 兑现方式平淡)。 |
| ❌ | **absent** (缺失) | 爽点强度 < 5/10;本集根本没有兑现承诺的爽点。几乎必然导致完播率崩塌 + 差评。 |

### 爽点放置的题材差异(2-3 示例)

不同题材的爽点放置有显著差异。以下是 3 个具体示例:

**Romance 爽点示例(女频虐恋):**
- 钩子锚定(0-3s):女主发现自己是替身。
- 第一次击中点(10-15s):男主承认"你只是个影子"。
- 中段升级(30-45s):女主默默准备离开。
- **爽点峰值(70%):女主在男主面前转身,丢下一句"我从来不是任何人的影子",然后坐进另一个男人的豪车。**
- 卡点(集末):男主追出来,但车已远去 —— 悬念:他会追多久?

**Revenge 爽点示例(男频战神归来):**
- 钩子锚定(0-3s):主角从私人飞机走下,接机大厅里曾经陷害他的人脸色骤变。
- 第一次击中点(10-15s):主角拒绝与对方握手,对方在媒体镜头前下不来台。
- 中段升级(30-45s):主角接管对方资金链的关键环节。
- **爽点峰值(75%):主角在董事会上让对方亲手签下转让协议,对方当场崩溃下跪。**
- 卡点(集末):主角接到电话 —— 一个更强大的对手出现了。

**Comedy 爽点示例(职场逆袭):**
- 钩子锚定(0-3s):新员工第一天撞了 CEO。
- 第一次击中点(10-15s):CEO 不怒反笑,说"你像我一个故人"。
- 中段升级(30-45s):新员工被同事排挤,但默默解决了一个 CEO 都没搞定的问题。
- **爽点峰值(72%):CEO 当众宣布"这个项目归她管",排挤她的同事脸色精彩。**
- 卡点(集末):CEO 单独留下她,说出一个秘密 —— 承诺下集揭穿。

## Conflict Type Vocabulary

阶梯式升级可以部署多种冲突原型。不同冲突类型解锁不同类型的爽点。以下是 5 类核心冲突原型:

| 冲突类型 | 定义 | 典型解锁的 爽点类型 |
|----------|------|---------------------|
| **内心冲突** (internal conflict) | 主角在两种价值观 / 情感 / 选择之间的挣扎(如:复仇 vs 原谅 / 爱情 vs 亲情)。 | 情感宣泄型爽点 🎯 —— 主角最终做出选择瞬间。 |
| **人际冲突** (interpersonal conflict) | 两个具体角色之间的对立(如:婆媳 / 闺蜜 / 兄弟 / 上下级)。 | 打脸型爽点 🎯 —— 一方被另一方公开碾压 / 揭穿。 |
| **阶级冲突** (class conflict) | 不同社会阶层之间的对立(如:豪门 vs 草根 / 富二代 vs 打工人)。 | 逆袭型爽点 🎯 —— 弱势一方反超强势一方。 |
| **道德冲突** (moral conflict) | 主角在"做正确的事"与"做有利的事"之间的抉择。 | 救赎型爽点 ✅ —— 主角选择正确道路后获得意外回报。 |
| **命运冲突** (fate conflict) | 主角与不可抗力(疾病 / 命运 / 时代)之间的对抗。 | 抗争型爽点 ✅ —— 主角虽然未能改变结局,但赢得了尊严。 |

### 阶梯可叠加多个冲突类型

一集 短剧不必只用一种冲突。优秀的阶梯通常叠加 2-3 种冲突类型 —— 例如:婆媳剧(人际冲突)叠加阶级冲突(豪门 vs 草根),叠加内心冲突(女主在"忍让"与"反击"之间的挣扎)。但叠加需谨慎:超过 3 种会导致主线混乱。

## Multi-Episode Escalation

阶梯式升级不仅适用于单集,还延伸到多集 短剧(尤其是 [小程序剧](../../_shared/glossary.md#小程序剧-mini-program-drama) 的 10-80 集长格式)。多集阶梯的核心规则:

### Episode N's 卡点 → Episode N+1's 钩子锚定

每一集的卡点 cliffhanger 必须直接驱动下一集的钩子锚定。这是"连续追看"的核心机制:

- **直接延续:** 第 N 集卡点"主角接到神秘电话" → 第 N+1 集 0-3s 钩子"电话那头的人是谁"。
- **时间跳跃:** 第 N 集卡点"三年后,主角归来" → 第 N+1 集 0-3s 钩子"归来的主角第一件事是做什么"。
- **视角切换:** 第 N 集卡点"反派决定动手" → 第 N+1 集 0-3s 钩子"反派动手的对象是谁"。

### Season-long Arc: 击中点 escalation curve

整季(典型 10-80 集)的击中点强度曲线应呈现"低 → 中 → 高 → 终局峰值"的弧形:

- **前期 (集 1-30%):** 击中点强度 6-7/10,主要是建立角色关系与冲突种子。
- **中期 (集 30-70%):** 击中点强度 7-8/10,冲突逐步升级,小爽点兑现。
- **后期 (集 70-90%):** 击中点强度 8-9/10,主冲突爆发,大爽点兑现。
- **终局 (集 90-100%):** 击中点强度 9-10/10,主线收束,终极爽点兑现。

### Serialized 卡点 Strategy

多集 短剧(尤其是付费 [小程序剧](../../_shared/glossary.md#小程序剧-mini-program-drama))的卡点策略遵循"硬卡点 + 软卡点"组合规则:

- **硬 卡点 (hard cliffhanger):** 每一集(尤其是付费集)结尾必须有强悬念卡点。典型强度:让"不看下一集就睡不着"。
- **软 卡点 (soft cliffhanger):** 集中段(约 40-50% 处)可选放置一个轻悬念,提升中段追看粘性。典型强度:让"想看但可以忍住"。

具体卡点放置规则(密度、强度分级、付费墙位置)由 [`paywall-design.md`](./paywall-design.md) (02-02) 定义。本 ref 只声明"硬+软组合"是阶梯式升级的多集延伸机制。

## Refresh Cadence

- **常规复审周期:** 每 90 天(从 `verified_date` 起算;下次复审日期 = 2026-09-15)。
- **责任方:** `hook_retention` 专家(无人工 owner —— 这是 skill 而非团队职责)。
