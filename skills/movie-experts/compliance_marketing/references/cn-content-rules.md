## AI 标识办法 (2025-09-01)

依据:网信办《人工智能生成合成内容标识办法》(国家互联网信息办公室令 第17号 / CAC Order No. 17),2025-09-01 起全网生效。适用于所有由 AI 生成或显著修改的、拟在中国大陆分发的内容(含文本、图像、音频、视频、交互式节目)。

### 显式标识 (Visible Mark)

**定义:** 在视频画面上叠加的可被人眼直接识别的 AI 来源标识,在播放全程持续可见。

| 维度 | 要求 | 备注 |
|------|------|------|
| 尺寸 | 标识高度 ≥ 画面高度的 5% | 竖屏 9:16 画面按短边方向计算;1080×1920 像素下标识高度 ≥ 96 像素 |
| 位置 | 右下角或左下角;不可被字幕条 / 安全区裁切 | 与字幕条不可重叠(字幕条通常占下方 15%) |
| 时长 | 全程持续显示 —— 不允许渐隐 / 淡出 | 与平台水印共存时,AI 标识必须视觉上可区分(不同颜色或不同区域) |
| 不透明度 | ≥ 70% | 防止被背景画面吞没 |
| 形式 | 文字 "AI 生成" 或图标 + 文字组合 | 纯图标不合规 —— 必须有可读文字 |

**示例叠加规格(竖屏 9:16,1080×1920):**
- 标识区域:`x=864..1056, y=1800..1900`(右下角,距底边 20px,距右边 24px)
- 文字:"AI 生成"(思源黑体 Regular,28px,白色 #FFFFFF)
- 描边:1px 黑色 #000000,不透明度 100%
- 背景:半透明黑色矩形,不透明度 70%

### 隐式标识 (Metadata)

**定义:** 嵌入文件元数据中的不可见标识,采用 C2PA / Content Credentials 风格字段,可被专业工具与平台审核系统机器读。

**必填字段(JSON-LD 片段示例):**

```json
{
  "@context": "https://schema.org",
  "@type": "MediaObject",
  "dc:creator": {
    "@type": "Organization",
    "name": "<出品方/制作方名称>"
  },
  "digi:source": "AI-generated",
  "digi:provenance": [
    {
      "step": 1,
      "action": "generation",
      "model": "<video_gen_primary>",
      "timestamp": "2026-06-15T08:30:00Z"
    },
    {
      "step": 2,
      "action": "edit",
      "operator": "editor expert",
      "timestamp": "2026-06-15T09:15:00Z"
    }
  ],
  "digi:ai_disclosure_present": true,
  "digi:visible_mark_height_ratio": 0.052
}
```

**关键字段说明:**
- `digi:source` 必须为 `"AI-generated"` 或 `"AI-assisted"`(后者用于真人主导 + AI 辅助)
- `digi:provenance` 为链式记录,每一道生成 / 剪辑 / 调色 / 配音工序都要追加一个 step
- `digi:ai_disclosure_present` 必须与显式标识的实际存在一致(false positive 是合规违规)
- 字段命名遵循 C2PA 1.4+ 规范(`digi:` 前缀对应 C2PA digital source 类型)

### 文本披露 (Script Disclosure)

**定义:** 在节目正文中以语音或字幕形式明确告知观众内容含 AI 生成成分。适用于含角色对白或旁白的短剧 / 漫剧。

**模板变体(按节目阶段):**

| 阶段 | 模板 | 适用场景 |
|------|------|----------|
| 冷开场 (cold-open) | `本作品由 AI 辅助生成,部分画面与声音经过人工智能处理` | 前 5 秒,与开场 logo 同步出现 |
| 片中 (mid-roll) | `(本场景由 AI 生成)` | 仅当某一场景为 100% AIGC 时插入;真人主导场景可省略 |
| 结尾卡片 (end-card) | `本作品由 <出品方> 联合 AI 创作工具制作,生成式 AI 使用情况已备案` | 片尾最后 3 秒,与署名同步 |

**强制条件:**
- 任何含 AI 生成角色面部、AI 生成语音、AI 生成背景的节目,必须至少有一处文本披露
- 文本披露字号 ≥ 字幕条字号的 80%(防止被压成"免责小字")
- 文本披露的停留时长 ≥ 2 秒(确保可读)

## AI 漫剧 备案 Regime (2026-04-01)

依据:广电总局《人工智能生成剧集(漫剧)网络播出管理试点办法》*estimated* (2026-04-01 生效)。"漫剧"指由 AI 生成或显著参与的、拟在视频平台分发的连续剧式内容。备案要求按内容类型 × 规模触发。

> ⚠️ 以下矩阵中的具体阈值(集数 / 时长 / 商业门槛)为基于公开征求意见稿与行业通报的 `*estimated*` 值。广电总局正式公告发布前,实际触发阈值以官方公告为准。本矩阵按 **"宁可多备,不可漏备"** 原则构造。

### 备案触发矩阵

| 内容类型 | 集数阈值 | 总时长阈值 | 商业意图 | 付费机制 | 备案要求 |
|----------|----------|------------|----------|----------|----------|
| 真人 + AIG(真人主演 + AI 辅助场景/特效) | ≥ 10 集 *estimated* | ≥ 60 分钟 *estimated* | 任何含广告 / 付费解锁 / 会员独享 | 付费解锁任何 1 集 → 触发 | 普通备案(提交剧本摘要 + AIG 使用清单) |
| 拟人化动画(全角色为 AI 生成动画形象) | ≥ 6 集 *estimated* | ≥ 30 分钟 *estimated* | 任何商业分发(含平台分账) | 平台分账即触发 | 普通备案 + 角色形象版权声明 |
| 全 AIG(画面/声音/剧本全 AI 生成) | ≥ 3 集 *estimated* | ≥ 15 分钟 *estimated* | 任何形式分发(含免费但带广告) | 任何 → 触发 | 重点备案(完整 AIG 流水日志 + 模型清单 + 训练数据来源声明) |
| 互动剧(分支剧情 / 用户选择驱动) | ≥ 1 集 *estimated*(互动内容默认重审) | ≥ 5 分钟 *estimated* | 任何 | 任何 → 触发 | 重点备案 + 互动分支逻辑说明书 + 未成年人保护评估 |

**备案流程关键节点(预估):**
1. **预审:** 提交剧本摘要 + AIG 使用范围声明(全 AIG / 真人 + AIG / 拟人化)
2. **技术审:** 提交显式 + 隐式标识样本;平台侧验证 metadata 可读性
3. **内容审:** 按 §1..§8 红线全量过审;互动剧需评估所有可达分支
4. **发放备案号:** 通过后节目携带备案号分发(类似网络剧备案号格式 `网剧备字(2026)第 XXXX 号` *estimated*)
5. **复备:** 节目内容大幅修改(单集修改 ≥ 30% 时长 *estimated*)需重新备案

**未备案后果(预估):** 平台下架 + 信用扣分 + 出品方 6 个月内禁止新节目备案 *estimated*。

## 8-Category 内容审核红线 Checklist

> 8 条红线为固定分类(来自 CONTEXT D-1,不可增减)。每条红线对应一个或多个具体法规条款。下游专家(screenplay / editor / drawer 等)在生成内容后必须由 `compliance_marketing` 专家按此 checklist 全量过审。

### §1 政治敏感

**定义:** 涉及国家领导人、国家象征、领土完整、政治制度、历史定论等敏感议题的内容。

**典型违规示例(短剧语境):**
- 模仿国家领导人形象 / 声音作为角色出场
- 涉及领土争议地图(如错误标注南海 / 藏南 / 钓鱼岛)
- 历史革命题材歪曲史实(如美化军阀、丑化革命先烈)

**触发关键词 / 画面信号:**
- 关键词:领导人姓名 + 模仿/扮演;革命 + 颠覆/嘲讽;领土名 + 争议/独立
- 画面信号:国旗倒置 / 撕毁;领导人人像特写;红头文件伪造特写

**严重等级:** 🔴 一律拒绝(任何疑似命中即拒,不可降级)

**引用法规:** 《网络安全法》第12条;《网络音视频信息服务管理规定》第10条;《互联网视听节目服务管理规定》第16条

### §2 暴力血腥

**定义:** 直接展示或详细描绘暴力行为、血腥伤口、虐待、酷刑、自残等引发强烈生理不适的内容。

**典型违规示例(短剧语境):**
- 复仇题材展示刑讯逼供的完整过程(超过 5 秒的特写)
- 武打戏使用真实刀具特写出血伤口(非特效血)
- 校园霸凌题材展示施暴完整过程(可作为剧情但不可作为爽点重复展示)

**触发关键词 / 画面信号:**
- 关键词:酷刑 / 凌迟 / 虐杀 / 自残 + 详细描写
- 画面信号:开放性伤口特写 > 2 秒;血液喷溅慢镜头;刀具刺入瞬间特写

**严重等级:** 🟡 情境依赖(剧情必需的暴力可降级处理 —— 切至后果镜头 / 暗示性剪辑;纯展示性暴力 = 🔴)

**降级方案:** 将暴力展示改为后果展示(如角色倒地后镜头切走,仅保留声音暗示);时长压缩至 ≤ 2 秒;避免特写开放性伤口。

**引用法规:** 《网络短视频内容审核标准细则》第3条;《未成年人节目管理规定》第9条(与 §4 联动)

### §3 色情低俗

**定义:** 含有性行为、性暗示、性器官暴露、低俗言语、引诱性内容等违反公序良俗的内容。

**典型违规示例(短剧语境):**
- 豪门虐恋题材展示完整性行为过程(即便穿衣服也违规)
- 女频"霸总强宠"中女性角色反复被脱衣 / 强吻的特写
- 旁白或台词含性器官隐晦指代("下面" + 行为描写)

**触发关键词 / 画面信号:**
- 关键词:做爱 / 上床 / 强奸 + 详细描写;性器官俗语 + 行为
- 画面信号:接吻镜头 > 5 秒且伴随脱衣动作;床戏镜头(即便盖被);裸露肩部以下的特写

**严重等级:** 🔴 一律拒绝(无降级空间 —— 性相关内容在短剧平台零容忍)

**引用法规:** 《网络短视频内容审核标准细则》第4条;《互联网视听节目服务管理规定》第16条;《广播电视管理条例》第32条

### §4 未成年人保护

**定义:** 涉及未满 18 周岁未成年人的内容,包括未成年人作为角色、未成年人作为目标观众、校园场景等。

**典型违规示例(短剧语境):**
- 校园题材展示未成年角色的恋爱 / 暴力 / 自杀情节
- 未成年人角色由 AI 生成面部(即便虚构也触发深度伪造红线)
- 萌宝题材让婴幼儿角色出现在危险场景(火灾、坠落、窒息)

**触发关键词 / 画面信号:**
- 关键词:未成年 / 学生 / 小学生 / 初中生 + 恋爱/暴力/性
- 画面信号:校服特写 + 暧昧镜头;未成年人面部特写(任何 AI 生成 = 违规);婴幼儿 + 危险动作

**严重等级:** 🔴 一律拒绝(未成年人 + AI 生成面部 = 不可降级;未成年人作为剧情角色进入危险/暧昧场景 = 🔴)

**降级方案(仅适用于剧情必需):**
- 校园题材必须将角色年龄明确为已成年(改写为大学 / 职场背景)
- AI 生成角色面部时禁止使用未成年人面孔模型
- 萌宝题材只展示婴幼儿在安全环境(成人怀抱 / 婴儿床)的镜头

**引用法规:** 《未成年人保护法》第50条、第74条;《未成年人节目管理规定》全文;《网络短视频内容审核标准细则》第11条

### §5 民族宗教

**定义:** 涉及民族关系、宗教信仰、民族风俗、宗教仪式等可能引发民族宗教矛盾的内容。

**典型违规示例(短剧语境):**
- 都市修仙题材歪曲道教 / 佛教教义为"超能力"
- 古装题材丑化特定民族的风俗(如错误描绘少数民族服饰)
- 涉及宗教场所(寺庙 / 清真寺 / 教堂)的亵渎性场景

**触发关键词 / 画面信号:**
- 关键词:佛/道/真主/上帝 + 亵渎/嘲讽;民族名 + 歧视/落后
- 画面信号:宗教场所内部 + 暧昧/暴力镜头;宗教法器作为道具被破坏

**严重等级:** 🟡 情境依赖(中性提及宗教 / 民族 = 🟢;歪曲、丑化、亵渎 = 🔴)

**降级方案:** 宗教元素仅作背景(远景寺庙外部);避免宗教教义的具体讨论;少数民族角色避免刻板印象(酒鬼 / 落后 / 野蛮)。

**引用法规:** 《宪法》第36条;《民族区域自治法》;《网络短视频内容审核标准细则》第6条、第7条

### §6 歧视侮辱

**定义:** 基于性别、地域、外貌、身体缺陷、职业、收入等维度对特定群体的歧视、侮辱、刻板印象强化。

**典型违规示例(短剧语境):**
- 男频题材反复将女性角色塑造为"拜金女 / 花瓶 / 工具人"
- 女频题材将农村角色 / 低收入角色塑造为"野蛮 / 粗鄙 / 笑料"
- 残疾角色作为"可怜 / 恐怖 / 突兀"的剧情工具

**触发关键词 / 画面信号:**
- 关键词:地域黑(河南人/东北人 + 贬义);外貌歧视(丑/胖 + 嘲笑);残疾 + 恐怖化
- 画面信号:特定群体被集体嘲笑的长镜头;外貌缺陷特写 + 滑稽配乐

**严重等级:** 🟡 情境依赖(单角色塑造 = 🟢;上升到群体性贬低 / 重复强化刻板印象 = 🔴)

**降级方案:** 反派个体的恶劣行为不外溢到其所属群体(一个拜金女角色 ≠ "女性都拜金");身体缺陷不作为笑点。

**引用法规:** 《民法典》第109条(人格权);《网络短视频内容审核标准细则》第8条;《广告法》第9条(与 §7 联动)

### §7 虚假宣传

**定义:** 含有虚假或引人误解的商业宣传内容,包括但不限于夸大功效、虚构疗效、伪造用户评价、诱导消费等。

**典型违规示例(短剧语境):**
- 短剧剧情中植入"神药 / 神器"的广告桥段(角色使用某产品后立即见效)
- 卡点解锁后展示"投资稳赚 / 包过考试 / 包治百病"的诱导信息
- 伪装成剧情的保健品 / 金融服务广告

**触发关键词 / 画面信号:**
- 关键词:包治 / 根治 / 神药 / 稳赚 / 包过;虚假用户证言("我用了一个月就...")
- 画面信号:产品包装特写 + 疗效夸张展示(白发变黑、立即瘦身);二维码 + "扫码领福利"叠加

**严重等级:** 🔴 一律拒绝(医疗、金融、教育类虚假宣传在短剧平台零容忍;普通商品的剧情植入 = 🟡 需明确广告标识)

**引用法规:** 《广告法》第4条、第9条、第17条、第25条;《反不正当竞争法》第8条;《网络短视频内容审核标准细则》第13条

### §8 版权侵权

**定义:** 未经授权使用他人享有著作权的作品,包括但不限于剧本、音乐、画面、角色形象、商标等。

**典型违规示例(短剧语境):**
- AI 生成的角色面部与某真实明星高度相似(深度伪造)
- 背景音乐使用未授权的流行歌曲(即便 5 秒片段也侵权)
- AI 生成的场景美术明显复刻某动漫 / 电影的标志性场景
- 台词直接引用某网络小说的原创段落(超过合理引用范围)

**触发关键词 / 画面信号:**
- 关键词:明星姓名 + 模仿/扮演;知名作品名 + 同人/复刻
- 画面信号:角色面部相似度 > 70% 与某真实公众人物 *estimated*;场景美术与某 IP 标志性画面相似度 > 60% *estimated
- 音频信号:背景音乐旋律相似度 > 8 小节 *estimated

**严重等级:** 🟡 情境依赖(原创内容 = 🟢;相似度达到上述阈值 = 🔴,需提供授权证明方可降级)

**降级方案:**
- 角色面部:使用风格化 / 卡通化处理,降低与真实人物的相似度
- 背景音乐:使用平台提供的免版权曲库(抖音 / 快手均有创作人曲库)
- 场景美术:避免 1:1 复刻,提取风格特征而非具体构图
- 台词引用:控制在合理引用范围(单处 ≤ 50 字 *estimated*,全文引用 ≤ 总字数 5% *estimated*),并标注出处

**引用法规:** 《著作权法》第10条、第24条(合理使用);《商标法》第57条;《网络短视频内容审核标准细则》第14条

## AIGC Labeling Workflow

以下为 `compliance_marketing` 专家对一集已完成的短剧 / 漫剧添加合规标识的标准流程。每一步必须按序执行,任一步缺失即视为不合规。

1. **判定 AIGC 范围**
   - 审查整集内容,标记每一处 AI 生成或 AI 显著修改的画面 / 声音 / 台词
   - 输出 `aigc_usage_report.json`:列出每个时间段的 AIGC 类型(全 AIGC / AI 辅助 / 真人主导)
   - 判定依据:全 AIGC(画面 + 声音 + 剧本均 AI 生成)= "AI-generated";真人主导 + AI 辅助 = "AI-assisted"

2. **添加显式标识 (Visible Mark)**
   - 按 `### 显式标识` 规格生成右下角叠加层(标识高度 ≥ 画面高度 5%,全程持续显示)
   - 验证:标识不被字幕条 / 安全区裁切;不透明度 ≥ 70%;含可读文字 "AI 生成"

3. **写入隐式标识 (Metadata)**
   - 按 `### 隐式标识` 的 JSON-LD 片段填充文件 metadata
   - `digi:source` 字段必须与步骤 1 的判定一致
   - `digi:provenance` 链必须记录所有生成 / 剪辑 / 调色 / 配音工序
   - 验证:使用 C2PA 兼容工具读回 metadata,确认字段完整可读

4. **插入文本披露 (Script Disclosure)**
   - 按 `### 文本披露` 模板选择冷开场 / 片中 / 结尾卡片中的至少一处
   - 任何含 AI 生成角色面部 / 语音 / 背景的节目必须至少一处文本披露
   - 验证:文本披露字号 ≥ 字幕条字号 80%,停留 ≥ 2 秒

5. **触发备案检查(仅适用于漫剧)**
   - 按 `## AI 漫剧 备案 Regime` 矩阵判定是否触发备案
   - 若触发:生成本集的备案材料清单(剧本摘要 + AIGC 使用清单 + 模型清单)
   - 若不触发:在 metadata 中记录免备依据(如"单集 + 免费 + 总时长 < 阈值")

6. **红线全量过审**
   - 按 §1..§8 逐条 checklist 过审,输出 `redline_review.json`
   - 任何 🔴 命中必须返工或拒绝分发;🟡 命中必须出具降级方案
   - 验证:8 条红线均有明确判定(通过 / 降级 / 拒绝)

7. **归档**
   - 将 `aigc_usage_report.json` + `redline_review.json` + 显式标识样本截图 + metadata dump 归档
   - 归档保留期 ≥ 节目下架后 2 年 *estimated*(配合监管事后审查)

## Refresh Cadence

- **常规复审周期:** 每 90 天(从 `verified_date` 起算,下次复审日期 = 2026-09-15)
- **责任方:** `compliance_marketing` 专家(无人工 owner —— 这是 skill 而非团队职责)
- **复审动作:**
  1. 检索网信办 + 广电总局官网是否有新通知 / 修订
  2. 核对所有 `*estimated*` 阈值是否有官方公告更新
  3. 更新 `Last-verified` + `verified_date` 时间戳
  4. 若阈值更新,同步修订 `## AI 漫剧 备案 Regime` 矩阵与各红线条目的具体数值
- **过期处理:** `scripts/verify_skill_references.py` 在 Last-verified > 90 天时会标记本 ref 为 stale,触发 audit 报警

## Drift Signals

以下事件触发本 ref 的 **非常规(提前)复审** —— 不必等到 90 天周期:

- **新通知发布:** 网信办 / 广电总局官网发布涉及 AI 生成内容 / 漫剧 / 短剧的新《办法》/《规定》/《通知》
- **备案政策更新:** 广电总局发布 AI 漫剧 备案 征求意见稿或正式办法修订
- **平台政策修订:** 抖音 / 快手 / 微信小程序 任意一方发布影响 AIGC 标识 / 备案 / 红线的平台规则修订
- **季末执法潮报告:** 季度末大量短剧 / 漫剧下架通报发布(通常每季度末),需复核本 ref 的红线阈值是否仍准确
- **红线新增信号:** 出现本 ref §1..§8 未覆盖的新型违规模式(如新型深度伪造、新型诱导消费手法),需评估是否扩展红线 checklist
- **C2PA / Content Credentials 规范更新:** 影响本 ref `### 隐式标识` 的字段命名与必填项

> 本 ref 的所有 `*estimated*` 标注在官方公告确认前,下游专家应按"宁严勿松"原则处理 —— 即按现有阈值合规,即便后续官方阈值更宽松。
